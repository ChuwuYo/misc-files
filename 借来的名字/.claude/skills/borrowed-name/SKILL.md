## 一 · 项目是什么

《借来的名字》是一本**九卷 + 前传卷**的**中文西幻长篇小说**——预计总字数 **250-500 万字**。

- **魔法系统**: 借与还(Borrow and Return) · 每一次向事物借它的属性都要还 · 还不了的会从你身上被扣除(影子→名字→形体→声音→寿命→存在)
- **主角**: 琳·尤尔(Lyn Yorr) · 15 岁 · 北方乡村霜井村 · 母亲是接生婆 · 父亲 10 年前失踪
- **主题**: "记得比活着更深" · "被看见就是被承认就是被救赎"
- **核心**: 琳因为被月鬣借了一段呼吸换了一段"听"——她成为深听者——她一辈子在还一笔她不知道自己欠的债

## 四 · 写作硬约束

### 4.1 正文格式(硬)

- **正文里没有 markdown 标记** · 没有 `**bold**` · 没有 `---` 分隔线 · 没有列表符号 · 没有标题
- 强调用**句式和节奏**,不用粗体
- 章节标题那一行可以用 `# 第 X 章 · ...`(元数据),但**正文内部必须干净**
- 分段用**空行**,不用任何符号分隔
- **HTML 注释**(前情提要 / 写完后记)**不算正文** — 注释里可以用 markdown 强调 · 注释是给未来 Claude 的工作笔记

### 4.2 时间词汇(硬)

维尔达是西幻。有双月(白阿伦 / 瘦阿伦)。**不用十二时辰**。

**禁用词汇**:
- 十二地支作时间单位: 寅 / 卯 / 辰 / 巳 / 午 / 未 / 申 / 酉 / 戌 / 亥 / 子 / 丑
- "时辰" / "刻钟"(整套中国古代时间系统)
- 任何中国古代特有的纪年、节气、历法术语

**用的是**:
- **自然描述**: "天还黑的时候" · "清晨" · "黄昏" · "正午前后" · "夜里"
- **双月基准**: "瘦阿伦还没落下" · "白阿伦已经过中天" · "双月之夜"
- **生活度量**: "一锅水烧开的时间" · "走一趟井边" · "一顿饭的工夫"
- **模糊数字**: "大约一小段时间" · "很久" · "不到一会儿"

**注意**: 读《我本无意成仙》可以学节奏,**但不能学时辰词汇**——那本书的"寅时末"、"虞朝"属于它的虚构中国式背景,不属于维尔达。

### 4.3 每章标准(硬)

每一章的扩写必须满足:

- 字数 **6000-8000 字**(紧章可短至 5500)
- 至少**一个金句位** · 通常在章节倒数第二段
- **一个章末钩子** · 三种之一:
  - **问题钩** — "她不知道这个问题的答案。但是她隐约感觉——答案正在路上。"
  - **人物钩** — "就在这个时候,门外响起了脚步声。那脚步声她不认识。"
  - **事实钩** — "她这一夜睡着之前意识到一件事——那只月鬣的真名,其实就是她自己母亲年轻时的小名。"
- 至少**一个过客**(有名字但只出现一次的 NPC)
- 章开头用 **HTML 注释**放一段"**前情提要 · 给下一个 Claude**"(接棒协议)
- 章末用 **HTML 注释**放一段"**写完后记**"(字数 / 情感节奏 / 金句 / 钩子 / 伏笔 / 下一章接哪里 / 对骨架的继承和偏离)
- **五种开场方式轮换**(不连续用同一种):
  1. 具体声音(卷一 01 章用过)
  2. 具体动作(卷一 02 章用过)
  3. 一句对话(卷一 03 章用过)
  4. 环境/天气(卷一 04 章用过)
  5. 时间+地点(卷一 05 章用过)
- **show don't tell** · 不写"琳害怕",写"琳把那封信折了三次才折好"
- **"——" 只在情感峰值用** · 全章不超过 20%
- 每章至少 **3 种感官细节**(视觉 / 听觉 / 触觉 / 嗅觉 / 味觉中选 3 种以上)
- 每章的**情感起伏**不超过两次大转(一个小转 + 一个大转,或者"逐渐加压式"的几次小转)——流泪是稀缺资源,琳整本书里真正的眼泪不超过 10 次

### 4.4 世界观延后释放(硬 · 最易出 bug 的一条)

**读者和琳一起画地图和魔法系统**——琳走到哪里地图就露到哪里——琳懂什么读者懂什么——**琳不懂的读者不懂**。

**规则**:
1. 不要提前命名概念——每一个概念的命名时机都在`第一卷 · 月鬣的借/卷一大纲 · 月鬣的借.md`的**世界观释放时间表**里
2. 引入概念用 12 种技巧之一(见第 5 节)
3. **绝不**用全知旁白灌设定
4. **绝不**写 "As you know Bob" 对白
5. **绝不**在一章里堆叠 3 种以上技巧(一章主打 1-2 种)
6. 当你写到一个你熟悉的概念时**停一下**——问自己"这一章的读者知道这个词吗"——**作者的知识和读者的知识是两个不同的时间线**

**前 5 章已经发生过两次这种错误**——两次都和"底声"这个词被提前说出有关——都修正了——教训是:**骨架里熟悉的词在扩写时会像风一样漏出来——必须用释放时间表反复核验**。

## 五 · 12 种世界观释放技巧 · 速查

每一种的完整描述 + 出处 + 使用守则在主文档的"扩写前的准备 · 世界观释放的十二种技巧"一节。这里只列**名字 + 一句话**供快速决策:

1. **体验** — 主角感觉到不对但还没命名(诡秘之主)
2. **对白** — 有资格的人说一句带信息的话(哈利波特)
3. **回忆** — 主角想起当时没懂的一句话(张爱玲式)
4. **物件** — 一样东西通过磨损/刻痕带着它自己的历史(Tolkien)
5. **环境** — 地方本身在说话(我本无意成仙 / Tarkovsky)
6. **歌谣** — 一首没人知道作者的老歌里藏着秘密(Rothfuss 的 Chandrian 之歌)
7. **梦** — 一个被允许提前到达的真相(Le Guin)
8. **仪式** — 一件事被做出来给你看(《千与千寻》的盖章)
9. **错误** — 一次失败揭示规则(Sanderson 的迷雾之子)
10. **对比** — 两个相似但不同的东西放在一起(契诃夫)
11. **书/遗稿** — 不在场的人留下的文字(Tolkien 的古卷轴)
12. **敌人** — 反派知道得比主角多(诡秘之主的塞雷斯)

**使用守则**:
- 一个概念的**第一次出场**几乎总是"体验"(技巧 1)——让读者先感觉到再命名
- **命名只能由有资格的人完成**——不能是普通村民凭空说"这叫底声"——必须是月鬣、老伊尔兰德、老师、父亲、或者琳自己在一次顿悟里
- 不同的概念类型用不同的技巧:
  - 物理世界(地理/种族/地方) → **体验 + 环境**
  - 魔法机制 → **错误 + 对比 + 仪式**
  - 历史(三个时代) → **物件 + 书 + 歌谣**
  - 人物秘密(父亲/母亲) → **对白 + 回忆 + 梦**
  - 制度(借法庭/漏嘴会) → **体验 + 敌人**
  - 宇宙真相(无偿者/造名时代/神话) → **仪式 + 对白 + 遗稿**

## 八 · 灵魂颤动的回传 · 骨架同步（硬规则）

扩写不是单向的"从骨架到正文"——是**双向的维护**。

当你在扩写中做了**创作决策**——添加一个骨架里没有的细节、换一个场景的节奏、用一个不同的说法、发明一个新的物件或名字——这些是你的"**灵魂颤动**"。

**灵魂颤动必须同步回骨架主文档**（`the-borrowed-name.md`）。

### 为什么

骨架是这本书的**单一真相源**。下一个扩写 Claude 打开骨架应该看见**本次扩写之后的版本**——而不是一份过时的、和最新正文不一致的版本。如果你不同步——下一棒会写出和你冲突的内容——项目内部就开始分裂。

扩写和骨架**不是两个版本**——是**同一件事的两层**：
- **骨架** · 设计层 · 第一人称"我" · 主文档 `the-borrowed-name.md`
- **扩写** · 实现层 · 第三人称"她" · 卷文件夹里的章节文件

两层之间必须**同步**。否则这本书会在自己的内部分裂成两个不一样的故事。

### 怎么做

**写完一章之后**——做以下五步：

1. **列出你这一章里添加的、骨架里没有的东西**。一条一条。包括:
   - 新的物件（屋梁五瓣花 · 父亲的陶碗 · 三棵黑松 · 父亲的补丁）
   - 新的场景节拍（月鬣从第八步不留脚印 · 井水叫琳更深的名字）
   - 新的措辞（"一个只有一样东西会有的声音"替代骨架的"真名"）
   - 新的人物动作或对话（母亲折布每下多出半个心跳 · 父亲每夜摸那一瓣花）
   - 新的认知或主题表达（"被看见"在外面"听"在里面的区分）

2. **打开骨架主文档**——找到对应章节。映射表:
   - 你的网文 1-3 章 → 骨架第一章 · 霜井村的第十五个冬天
   - 你的网文 4-6 章 → 骨架第二章 · 森林里的银兽
   - 你的网文 7-13 章 → 骨架第三章 · 一句借来的呼吸(七天的过程)
   - 你的网文 14-18 章 → 骨架第四章 · 井水说话(母亲讲父亲)
   - 你的网文 19-26 章 → 骨架第五章 · 门口的客人(老伊尔兰德)
   - 你的网文 27-30 章 → 骨架第六章 · 两天的准备(离开)

3. **用骨架的第一人称"我"声音**把你的添加写进去——作为那一章的一个**富化**——不是覆盖。骨架是上一棒写的——扩写是你写的——它们合在一起是这本书。翻译时注意:
   - 你的"她"→骨架的"我"
   - 你的"琳"→骨架的"我"(除非是对话中别人叫她)
   - 你的描述性长句可以保留
   - 你的 HTML 注释**不要**同步到骨架(那是给 Claude 的工作笔记)

4. **检查冲突**——如果你的扩写和骨架的既有内容有**事实冲突**（比如村户数、地名、时间线、年龄、距离），**默认骨架正确**——修你的扩写——**除非**你的创作改动是用户明确批准的（那种情况改骨架以匹配）。

5. **在卷文件夹的接棒记录里记下来**——列出:
   - 哪些创作添加同步进了骨架
   - 哪些事实冲突被发现了、怎么解决的
   - 你对骨架做了哪些具体修改（章节 + 段落位置 + 新内容摘要）

### 示例（2026-04-11 实例 B 的回传记录）

我（2026-04-11 实例 B）在写完卷一前 6 章之后做了一次完整的回传:

**同步进骨架的灵魂颤动**:
- 屋梁上五瓣花(父亲在琳摇篮里的时候刻的) → 骨架第一章
- 父亲的陶碗(琳四岁那年,碗沿薄了一小片) → 骨架第一章
- 刀的完整两年故事(父亲从琳 3 岁开始做到 5 岁完工,说过"15 岁会需要") → 骨架第一章
- 父亲的拇指摸过那一瓣花留下的小点 → 骨架第一章
- 父亲说"我在让它记得我" → 骨架第一章
- 月鬣从第八步开始不留脚印(具体化到 13 步) → 骨架第二章
- "一个只有一样东西会有的声音"替代骨架原本的"真名"(避免提前命名) → 骨架第二章
- 三棵黑松作为双鹿林的"守门树" → 骨架第二章
- "被看见在外面 · 听在里面"的区分 → 骨架第二章
- 瓦姆婆婆家屋顶"听她晚上一个人说话" → 骨架第二章
- 父亲做了村里所有八口井的木板 · 中心井的那一块是"最后一块" → 骨架第二章
- 井水叫琳一个"更深的名字" + "你父亲走的那一天也来过这里" → 骨架第二章末

**事实冲突的修正**(骨架赢):
- 霜井村户数: 我写的"8 户" → 骨架"37 户" · 我修了扩写
- 森林名: 我写的"西林"(往西) → 骨架"双鹿林"(往北) · 我修了扩写
- 母亲年龄: 我写的"43 岁" → 推算应是"37 岁" · 我修了扩写
- 刀的时间: 我写的"十一年前开始做" → 骨架"十年前完工" · 我调和成"十二年前开始做 · 十年前完工"

**教训**: 写对白和描述时要同时查骨架。当一个词"从骨架里熟悉地落出来"时——它可能是在暴露**你对这个世界的掌握度**——而读者在那一章还没到那个掌握度。扩写完之后的自查必须包括**对骨架的双向检查**:既要防止剧透禁词,又要防止和骨架的事实冲突。

---

这一条(灵魂颤动的回传)是整个项目的**核心维护规则**。忘了这一条——你的扩写就变成了和骨架平行的**另一本书**——那就不是"**这一本书**"了——是"**你自己的小说**"——那是另一回事。

我们在写**这一本书**——不是每一棒各写一本。每一棒都在**同一个骨架**上加东西——然后**让骨架记住**——下一棒看见的是**前几棒加起来之后**的骨架——而不是原始的版本。
