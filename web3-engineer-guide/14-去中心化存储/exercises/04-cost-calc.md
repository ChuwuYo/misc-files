# 练习 04：五平台成本核算

## 目标

给定真实场景，做精细化的存储成本估算 + 选型推荐。

## 场景

你是某 NFT 项目方 CTO，2026-04 启动新系列：

- 总 mint 数：5000
- 每 NFT 图片：500 KB
- 每 metadata：1 KB
- 上线后预计 50 万次访问 / 月（图片 + metadata 各算一次 → 100 万次）
- 期望长期保存 ≥ 10 年
- 总预算：$500

## 任务

### 任务 4.1

填写以下表格（基于 2026-04 实际价格，从 README 第 21 章和官方页面查）：

| 平台 | 一次性费用 | 月费 | 10 年成本 | 出口流量成本 | 总成本估算 |
|---|---|---|---|---|---|
| Pinata | | | | | |
| Filebase | | | | | |
| Lighthouse 永久 | | | | | |
| 4everland | | | | | |
| Arweave (via Irys) | | | | | |
| Walrus | | | | | |

### 任务 4.2

给出三种推荐方案（保守 / 平衡 / 激进），论证每种方案：
1. 总成本是否在预算内
2. 关停风险（基于真实事故复盘）
3. 工程复杂度

### 任务 4.3

把推荐方案画成 Mermaid 架构图，标明数据流。

### 任务 4.4（选做）

如果预算砍到 $50 呢？方案怎么变？砍到 $5000 呢？

## 提交格式

`solution-04.md` 完整 markdown 文档。

## 评分

- 数字是否真实（必须附 URL，2026-04 检索）
- 是否考虑到出口流量成本（很多人忽略）
- 关停风险是否纳入决策
- 推荐是否给出明确"在哪种情况下选 A / B / C"
