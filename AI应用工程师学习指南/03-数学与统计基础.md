## 3.1 线性代数：Transformer 在做的事 90% 是矩阵乘法

> **速记三条**：
> 1. **向量是一列数、矩阵是一张数表、张量是一摞数表**——LLM 里所有东西最后都长这个样子。
> 2. **点积测"两个向量像不像"**：同方向数值大、垂直为 0、反方向为负。RAG 检索的相似度、attention 里的"该看哪儿"全靠它。
> 3. **`nn.Linear(in, out)` 内部就是一次矩阵乘法**——把 `in` 维向量映射到 `out` 维。Transformer 90% 的算力都花在这件事上。
>
> 3.1.5（特征值）、3.1.6（SVD）"知道有这回事就行"——理解 LoRA 时再回来看也来得及。
>
> → A 档读到这里可停（速记三条已涵盖工作必需）；B 档继续看正文 + 代码、公式当"看图说话"；C 档自己推一遍每个公式。

### 3.1.1 向量、矩阵、张量

向量是一列数：$\mathbf{x} = [x_1, x_2, \ldots, x_d]^T \in \mathbb{R}^d$。在 LLM 里，一个 token 经过 embedding 后就是一个 $d$ 维向量（GPT-2 small 是 768 维，Llama-3 8B 是 4096 维）。

矩阵是数的二维表：$\mathbf{A} \in \mathbb{R}^{m \times n}$。`nn.Linear(in_features=768, out_features=2048)` 的内部权重就是一个 $2048 \times 768$ 的矩阵。

张量（tensor）是任意维度的推广。在工程里你最常碰到的形状是 `(batch, seq_len, hidden_dim)`，三维张量。Vision 里多一维 `(batch, channels, height, width)`。

```python
import torch

# 一个 batch 的 token embedding：8 条序列、每条 128 个 token、每 token 768 维
x = torch.randn(8, 128, 768)
print(x.shape)          # torch.Size([8, 128, 768])
print(x.ndim)           # 3
print(x.numel())        # 786432，张量里的标量个数
```

记一个工程习惯：调试维度永远 `print(t.shape)`，比 `print(t)` 有用一百倍。

### 3.1.2 点积、范数、夹角

两个 $d$ 维向量 $\mathbf{a}, \mathbf{b}$ 的点积是

$$\mathbf{a} \cdot \mathbf{b} = \sum_{i=1}^{d} a_i b_i = \|\mathbf{a}\| \|\mathbf{b}\| \cos\theta$$

直觉：点积测量两个向量"有多大程度指向同一边"。同向点积大，正交点积为 0，反向点积为负。

L2 范数 $\|\mathbf{a}\| = \sqrt{\sum a_i^2}$ 是向量的长度。把向量除以自己的范数得到单位向量，方向不变长度为 1，叫归一化（normalize）。

cosine 相似度就是先归一化再点积：

$$\text{cos}(\mathbf{a}, \mathbf{b}) = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \|\mathbf{b}\|}$$

（代码对应：分子 `a @ b`、分母 `np.linalg.norm(a) * np.linalg.norm(b)`；PyTorch 一行版 `F.cosine_similarity(a, b, dim=-1)`。）

```python
import numpy as np

a = np.array([1.0, 2.0, 3.0])
b = np.array([2.0, 1.0, 0.0])

dot = a @ b                                  # 4.0
cos = (a @ b) / (np.linalg.norm(a) * np.linalg.norm(b))  # 0.2673
```

**对应到 LLM 哪儿**：RAG 的向量检索默认用 cosine 相似度，因为 sentence-transformers 系列的 embedding 已经归一化，cosine 等价于点积，但点积更快——FAISS 内部就是 `IndexFlatIP`（inner product）。OpenAI 的 `text-embedding-3-small` 文档明确说返回的是单位向量，可以直接点积。

为什么不用欧氏距离？对单位向量，$\|\mathbf{a} - \mathbf{b}\|^2 = 2 - 2\mathbf{a} \cdot \mathbf{b}$，和点积单调相关，结果一样但多一步计算。对未归一化的向量，欧氏距离会被向量长度主导，"长向量被压在搜索结果末尾"，这往往不是你想要的。

### 3.1.3 矩阵乘法：把 Linear 层看清楚

$\mathbf{C} = \mathbf{A}\mathbf{B}$ 中 $\mathbf{A} \in \mathbb{R}^{m \times k}$、$\mathbf{B} \in \mathbb{R}^{k \times n}$，结果 $\mathbf{C} \in \mathbb{R}^{m \times n}$，元素

$$C_{ij} = \sum_{p=1}^{k} A_{ip} B_{pj}$$

把它读成"$\mathbf{C}$ 的第 $i$ 行第 $j$ 列 = $\mathbf{A}$ 的第 $i$ 行点积 $\mathbf{B}$ 的第 $j$ 列"。

PyTorch 的 `nn.Linear(in_dim, out_dim)` 计算 $\mathbf{y} = \mathbf{x}\mathbf{W}^T + \mathbf{b}$，注意是 $\mathbf{W}^T$（PyTorch 的 weight 形状是 `(out_dim, in_dim)`）。这个细节经常坑 reshape：

```python
import torch.nn as nn

linear = nn.Linear(768, 2048)
print(linear.weight.shape)   # torch.Size([2048, 768])
print(linear.bias.shape)     # torch.Size([2048])

x = torch.randn(8, 128, 768)
y = linear(x)
print(y.shape)               # torch.Size([8, 128, 2048])
```

batched matmul 是工程主力。`torch.matmul` / `@` 在最后两维做矩阵乘，前面的维度按 broadcast 处理：

```python
Q = torch.randn(8, 12, 128, 64)   # (batch, heads, seq, head_dim)
K = torch.randn(8, 12, 128, 64)
scores = Q @ K.transpose(-2, -1)  # (8, 12, 128, 128)
```

**对应到 LLM 哪儿**：上面这段就是 multi-head attention 里 $QK^T$ 的样子。一个 forward 里 attention 模块的 FLOPs 主要花在三处矩阵乘：$QK^T$、$\text{softmax}(\cdot) V$、output projection，再加上 FFN 的两个矩阵乘（升维 $\times 4$ 再降回来）。"Transformer 的 90% 算力是矩阵乘法"不是夸张。

### 3.1.4 外积、Hadamard 积与广播

外积（outer product，符号 $\otimes$ 读作"克罗内克积"或干脆叫"外积"）：$\mathbf{a} \otimes \mathbf{b} = \mathbf{a} \mathbf{b}^T$，$\mathbf{a} \in \mathbb{R}^m, \mathbf{b} \in \mathbb{R}^n$，结果是 $m \times n$ 矩阵。直觉：拿两根细向量"撑开"成一张矩阵。LoRA 的核心 $\Delta \mathbf{W} = \mathbf{B}\mathbf{A}$ 在秩 1 时就是一个外积；秩 $r$ 是 $r$ 个外积之和。

Hadamard 积（逐元素乘，符号 $\odot$ 读作"哈达玛积"）：$(\mathbf{A} \odot \mathbf{B})_{ij} = A_{ij} B_{ij}$，要求形状相同。直觉：两个同样形状的矩阵摆在一起，对应位置相乘。在 PyTorch 里就是 `A * B`（注意 `*` 不是矩阵乘法，矩阵乘法是 `@` 或 `torch.matmul`，新人最容易踩这个坑）。GLU、SwiGLU 这类门控激活就是 $\sigma(\mathbf{W}_1 \mathbf{x}) \odot (\mathbf{W}_2 \mathbf{x})$。

广播（broadcasting）让小张量"自动复制"匹配大张量：`(B, T, D) + (D,)` 时第二项被当作 `(1, 1, D)` 在前两维复制。这是 numpy / PyTorch 工程的核心便利，但也是形状 bug 的高发地：`(N, 1) - (1, N)` 会变成 `(N, N)` 而不是元素差。养成 `assert x.shape == ...` 的习惯能省一半时间。

### 3.1.5 特征值与特征向量

矩阵 $\mathbf{A}$ 作用在向量上一般会改变向量的方向。但有些向量被 $\mathbf{A}$ 作用后只缩放不转向：

$$\mathbf{A}\mathbf{v} = \lambda \mathbf{v}$$

这样的 $\mathbf{v}$ 叫特征向量、$\lambda$ 叫特征值。直觉：特征向量是矩阵的"骨架方向"，特征值告诉你这个方向上是拉伸（$|\lambda|>1$）还是压缩。

工程师不必背特征值多项式，但要知道：协方差矩阵的特征向量给出数据的主方向（PCA 的根），雅可比矩阵的特征值告诉你迭代是否收敛（爆炸/消失梯度的根）。

```python
A = np.array([[3, 1], [0, 2]], dtype=float)
eigvals, eigvecs = np.linalg.eig(A)
print(eigvals)               # [3. 2.]
print(eigvecs)               # 每列是一个特征向量
```

### 3.1.6 SVD 与 PCA：让 1024 维 embedding 画在二维图上

任何矩阵 $\mathbf{A} \in \mathbb{R}^{m \times n}$ 都能分解为

$$\mathbf{A} = \mathbf{U}\mathbf{\Sigma}\mathbf{V}^T$$

$\mathbf{U}, \mathbf{V}$ 列向量正交，$\mathbf{\Sigma}$ 对角线上是奇异值（按从大到小排列）。前 $k$ 个奇异值对应的子空间是 $\mathbf{A}$ 信息量最大的 $k$ 个方向。

PCA 是把数据中心化后做 SVD，取前 $k$ 个主方向。最常用的工程场景是 embedding 可视化：把 1024 维降到 2 维画散点图。

```python
import numpy as np
import matplotlib.pyplot as plt

# 假装这是 200 个句子的 1024 维 embedding
X = np.random.randn(200, 1024)
X[:100] += np.array([3.0] + [0.0]*1023)  # 前 100 条偏向第一维

X_centered = X - X.mean(axis=0)
U, S, Vt = np.linalg.svd(X_centered, full_matrices=False)
X_2d = X_centered @ Vt[:2].T   # 取前 2 个主方向

plt.scatter(X_2d[:100, 0], X_2d[:100, 1], label='cluster A')
plt.scatter(X_2d[100:, 0], X_2d[100:, 1], label='cluster B')
plt.legend(); plt.title('PCA of 1024-d embeddings')
plt.savefig('pca.png')
```

**对应到 LLM 哪儿**：LoRA（Low-Rank Adaptation）的核心假设是"权重更新矩阵的有效秩很低"，于是把 $\Delta \mathbf{W} \in \mathbb{R}^{d \times d}$ 近似成 $\mathbf{B}\mathbf{A}$，其中 $\mathbf{B} \in \mathbb{R}^{d \times r}, \mathbf{A} \in \mathbb{R}^{r \times d}$，$r \ll d$。这正是 SVD 截断的工程化。一个 7B 模型用 LoRA 微调，可训练参数从 70 亿降到几百万，靠的就是低秩假设。

t-SNE / UMAP 是 PCA 之外更常用的 embedding 可视化方法，但它们是非线性的，不在线性代数范围内，留到后面章节。

## 3.2 微积分与自动微分：反向传播在算什么

> **速记三条**：
> 1. **梯度 = 函数上山最快的方向**，负梯度 = 下山最快。训练 = 往负梯度方向挪一小步。
> 2. **`loss.backward()` 在做的事就是从 loss 出发倒着把每一层"该挪多少"算出来**——这叫反向传播，本质是高中"复合函数求导"链式法则的张量版本。
> 3. **你永远不需要手算梯度**——PyTorch 自动微分（autograd）替你做。但你得会读「梯度爆炸」「梯度消失」「dead ReLU」这几个症状名词。
>
> 3.2.3（Jacobian / Hessian）和 3.2.4（手写 autograd）"看一遍有印象就行"，跳过不影响后面 PyTorch 实战。
>
> → A 档读到这里可停（记住"梯度 = 上山方向"+`loss.backward()` 替你算就够）；B 档继续看正文 + 代码、公式当"看图说话"；C 档把 3.2.2 的链式法则自己推一遍。

### 3.2.1 偏导与梯度

单变量导数 $f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}$ 你大概还记得（直觉：$x$ 动一点点，$f$ 跟着动多少的"动多少 / 动一点点"的比值）。多变量函数 $f(x_1, \ldots, x_n)$ 对每个变量**单独**求导（其他变量当作常数）得到偏导数 $\frac{\partial f}{\partial x_i}$（读作"$f$ 对 $x_i$ 的偏导"，符号 $\partial$ 是把 $d$ 写成弯的，专门标记"偏"），把它们堆成向量叫梯度：

$$\nabla f(\mathbf{x}) = \left[\frac{\partial f}{\partial x_1}, \ldots, \frac{\partial f}{\partial x_n}\right]^T$$

梯度的几何含义是"$f$ 增长最快的方向"。负梯度方向就是下降最快的方向。这是梯度下降的全部直觉。

```python
import torch

x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
f = (x ** 2).sum()        # f = 1 + 4 + 9 = 14
f.backward()
print(x.grad)             # tensor([2., 4., 6.])  即 2x
```

### 3.2.2 链式法则：backward 的本质

复合函数 $h(x) = f(g(x))$ 的导数是

$$h'(x) = f'(g(x)) \cdot g'(x)$$

多变量版本：如果 $\mathbf{y} = g(\mathbf{x})$、$z = f(\mathbf{y})$，那么

$$\frac{\partial z}{\partial \mathbf{x}} = \frac{\partial z}{\partial \mathbf{y}} \cdot \frac{\partial \mathbf{y}}{\partial \mathbf{x}}$$

后者是 Jacobian 矩阵 $\mathbf{J}_{ij} = \partial y_i / \partial x_j$。整张神经网络就是一长串复合函数 $\mathcal{L} = \ell(f_L(f_{L-1}(\cdots f_1(\mathbf{x}))))$，链式法则把每一层的局部梯度乘起来传回去——这就是反向传播。

直观说：每个算子知道"自己输出对自己输入的局部梯度长啥样"，autograd 引擎从 loss 出发，把 upstream 梯度一路乘下去。所有现代框架（PyTorch、JAX、TensorFlow）都把这个过程自动化了，工程师写 forward，框架记 graph，`backward()` 一调梯度全有。

手算一个最小例子。考虑两层 MLP：

$$\mathbf{h} = \mathrm{ReLU}(\mathbf{W}_1 \mathbf{x}), \quad \hat{y} = \mathbf{w}_2^T \mathbf{h}, \quad \mathcal{L} = (\hat{y} - y)^2$$

链式反推：

$$\frac{\partial \mathcal{L}}{\partial \hat{y}} = 2(\hat{y} - y), \quad \frac{\partial \hat{y}}{\partial \mathbf{w}_2} = \mathbf{h}, \quad \frac{\partial \hat{y}}{\partial \mathbf{h}} = \mathbf{w}_2$$

ReLU 的导数是指示函数（输入正为 1、否则为 0）：

$$\frac{\partial \mathbf{h}}{\partial \mathbf{W}_1 \mathbf{x}} = \mathrm{diag}(\mathbb{1}[\mathbf{W}_1 \mathbf{x} > 0])$$

把这些拼起来就能算 $\partial \mathcal{L} / \partial \mathbf{W}_1$。这就是 PyTorch 在你按下 `loss.backward()` 时做的事，只是它用张量批量化、并行化、并把每个算子的反向 closure 缓存起来。理解到这层，你就能解释为什么把 ReLU 换成"输入永远负"的死神经元（dead ReLU）会让 $\partial \mathbf{h} / \partial \cdot$ 全部为 0、梯度断裂，进而那一片参数永远不更新。

### 3.2.3 Jacobian 与 Hessian

向量到向量的函数 $\mathbf{f}: \mathbb{R}^n \to \mathbb{R}^m$ 的一阶导数是 Jacobian，$m \times n$ 矩阵。

标量函数的二阶导数推广是 Hessian $\mathbf{H}_{ij} = \frac{\partial^2 f}{\partial x_i \partial x_j}$，$n \times n$ 对称矩阵。Hessian 告诉你损失曲面的曲率：所有特征值大于零意味着这是局部极小（碗状），有正有负是鞍点。

工程实践里你**不会**直接算 Hessian——一个 7B 模型的 Hessian 是 $7 \times 10^9$ 行 $\times$ 列的矩阵，存不下。但 Hessian 的"近似形状"决定了二阶优化器（Newton、L-BFGS）的可行性，也是 Adam 这类自适应方法的灵感来源（用梯度平方的指数滑动平均近似对角 Hessian）。

### 3.2.4 自动微分实操

写一个最小可用的标量自动微分，理解一下框架是怎么记 graph 的：

```python
class Var:
    def __init__(self, value, parents=(), op=''):
        self.value = value
        self.grad = 0.0
        self.parents = parents
        self.op = op
        self._backward = lambda: None

    def __add__(self, other):
        out = Var(self.value + other.value, (self, other), '+')
        def _back():
            self.grad += out.grad
            other.grad += out.grad
        out._backward = _back
        return out

    def __mul__(self, other):
        out = Var(self.value * other.value, (self, other), '*')
        def _back():
            self.grad += other.value * out.grad
            other.grad += self.value * out.grad
        out._backward = _back
        return out

    def backward(self):
        topo, visited = [], set()
        def build(v):
            if v in visited: return
            visited.add(v)
            for p in v.parents:
                build(p)
            topo.append(v)
        build(self)
        self.grad = 1.0
        for v in reversed(topo):
            v._backward()


a = Var(2.0); b = Var(3.0)
c = a * b + a              # c = 2*3 + 2 = 8
c.backward()
print(a.grad, b.grad)      # 4.0 2.0   即 ∂c/∂a = b+1 = 4, ∂c/∂b = a = 2
```

这就是 PyTorch autograd 的简化版：每个算子注册前向值和反向 closure，反向时按拓扑序累加梯度。理解这一段，PyTorch 的 `register_hook`、`retain_grad`、`detach` 几个 API 一下就清楚了。

**对应到 LLM 哪儿**：训练 LLM 时一次 `loss.backward()` 会触发上万亿次 chain rule 乘法。混合精度（fp16/bf16）省的是矩阵乘的内存与算力；gradient checkpointing 省的是中间激活的存储——它通过"反向时重算 forward"换显存。这些技巧的可行性，全建立在"链式法则可以局部化"这个数学事实上。

## 3.3 概率与统计：模型在估计什么

> **速记三条**：
> 1. **LLM 输出的不是一个 token，而是"全词表上的一个概率分布"**——比如词表 5 万个词，模型给你 5 万个概率值，加起来等于 1。下一个 token 是从这个分布里抽出来的。
> 2. **训练 = 让模型给"正确答案"的概率尽量高**。这就是"最大似然估计 MLE"，所有分类 loss（包括 LLM 的 cross-entropy）都是这个目标的不同写法。
> 3. **L2 正则 = 给参数一个"以 0 为中心的高斯先验"**——这是贝叶斯视角，知道这层等价关系，调正则系数时心里有谱。
>

### 3.3.1 随机变量与分布

随机变量 $X$ 是把"不确定的结果"映射到数的函数。离散 RV 有概率质量函数 PMF：$P(X = x_i) = p_i$，且 $\sum p_i = 1$。连续 RV 有概率密度函数 PDF $f(x)$，$\int f(x) dx = 1$，注意 PDF 在某点的值不是概率本身，是密度。

期望（平均值）：$\mathbb{E}[X] = \sum_i x_i p_i$ 或 $\int x f(x) dx$。

方差：$\mathrm{Var}(X) = \mathbb{E}[(X - \mathbb{E}[X])^2]$。开根是标准差。

```python
import numpy as np
samples = np.random.randn(10000)   # 标准正态
print(samples.mean(), samples.std())   # 接近 0 和 1
```

### 3.3.2 工程师最常撞见的几种分布

| 分布 | 用在哪 | 关键参数 |
| --- | --- | --- |
| 伯努利 $\text{Bern}(p)$ | 二分类（点击 / 不点击）| 成功概率 $p$ |
| 类别 $\text{Cat}(\boldsymbol{p})$ | 多分类、token 采样 | 概率向量 $\boldsymbol{p}$ |
| 多项 $\text{Multinomial}$ | 多次类别试验的计数 | $n, \boldsymbol{p}$ |
| 正态 $\mathcal{N}(\mu, \sigma^2)$ | 权重初始化、噪声 | 均值 $\mu$、方差 $\sigma^2$ |
| Dirichlet $\text{Dir}(\boldsymbol{\alpha})$ | 类别分布的先验、LDA、RLHF | $\boldsymbol{\alpha}$ |

LLM 解码就是反复从 $\text{Cat}(\boldsymbol{p})$ 采样，$\boldsymbol{p}$ 由模型 logits 经过 softmax 得到。这是后面 3.7 节的根。

正态分布的 PDF：

$$f(x) = \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(x - \mu)^2}{2\sigma^2}\right)$$

工程上记一句话："均值定中心，方差定宽度"。Xavier / Kaiming 初始化都是从正态（或均匀）分布采样权重，方差由扇入扇出决定，目的是保持各层激活方差稳定。

### 3.3.3 联合、边缘、条件

$P(X, Y)$ 是联合分布；对 $Y$ 求和（积分）得边缘 $P(X) = \sum_y P(X, Y = y)$；条件 $P(X|Y) = P(X, Y) / P(Y)$。

独立的两个 RV 满足 $P(X, Y) = P(X) P(Y)$，等价 $P(X | Y) = P(X)$。"知不知道 Y 都不影响 X 的分布"。机器学习的"独立同分布（IID）假设"指的是训练样本之间互相独立、和测试样本来自同一分布。这个假设破裂时（covariate shift、concept drift）模型在线上就会崩。

### 3.3.4 贝叶斯：先验、似然、后验在 ML 里的样子

把贝叶斯定理读三遍：

$$\underbrace{P(\theta \mid D)}_{\text{后验}} = \frac{\overbrace{P(D \mid \theta)}^{\text{似然}} \cdot \overbrace{P(\theta)}^{\text{先验}}}{\underbrace{P(D)}_{\text{证据}}}$$

工程里的对应：

- 先验 $P(\theta)$：你在没看到数据前对参数的信念。L2 正则等价于"参数服从均值为 0 的高斯先验"——参数离 0 越远先验密度越小，惩罚越大。L1 正则等价于拉普拉斯先验，鼓励稀疏（权重直接为 0）。
- 似然 $P(D \mid \theta)$：参数 $\theta$ 下观测到数据 $D$ 的概率。
- 后验 $P(\theta \mid D)$：看完数据后更新的信念。MAP 就是找让后验最大的 $\theta$。
- 证据 $P(D)$：归一化常数，对优化 $\theta$ 不重要。

最大似然估计（MLE）找 $\theta$ 让 $P(D|\theta)$ 最大；最大后验（MAP）多一个先验项 $\log P(\theta)$。把 $\log P(\theta) = -\frac{\|\theta\|^2}{2\sigma^2} + C$ 代入就能看到："MAP = MLE + L2 正则"在数学上是同一回事，正则化系数对应先验的方差倒数。先验越窄（$\sigma$ 小）正则越强。

举个具体例子：垃圾邮件分类。$P(\text{spam} \mid \text{words})$ 算不出来，但用贝叶斯：

$$P(\text{spam} \mid \text{words}) \propto P(\text{words} \mid \text{spam}) P(\text{spam})$$

朴素贝叶斯进一步假设词与词独立 $P(\text{words} \mid \text{spam}) \approx \prod_i P(w_i \mid \text{spam})$。这个假设不对（"免费"和"领取"显然相关），但工程上够用。

LLM 时代你直接看到贝叶斯思维的地方：

- RAG 里的 query reformulation。先验是模型对答案的猜测，似然是检索到的文档支持度，后验是 grounding 之后的回答。
- prompt engineering 里的 in-context learning：先验是模型预训练得到的分布，few-shot 例子提供"似然"信号，后续 token 的预测分布相当于"近似后验"。
- LLM 自己作为先验，加上工具调用结果作为观测，迭代修正答案。

### 3.3.5 最大似然估计：Loss 的根

监督学习的标准故事：数据 $\{(\mathbf{x}_i, y_i)\}_{i=1}^N$ 假设独立同分布，模型给条件分布 $P(y | \mathbf{x}; \theta)$。似然

$$L(\theta) = \prod_{i=1}^N P(y_i | \mathbf{x}_i; \theta)$$

取负对数（连乘变累加，数值稳定）：

$$-\log L(\theta) = -\sum_{i=1}^N \log P(y_i | \mathbf{x}_i; \theta)$$

这就是负对数似然损失（NLL）。分类任务里把模型输出做 softmax 当作 $P(y|\mathbf{x};\theta)$，NLL 就是 cross entropy loss，下一节会推。

```python
import torch
import torch.nn.functional as F

# 模拟分类：3 个样本、5 类
logits = torch.randn(3, 5, requires_grad=True)
targets = torch.tensor([0, 2, 4])

loss_a = F.cross_entropy(logits, targets)
# 等价于
log_probs = F.log_softmax(logits, dim=-1)
loss_b = F.nll_loss(log_probs, targets)
print(torch.allclose(loss_a, loss_b))   # True
```

`F.cross_entropy` = `F.log_softmax` + `F.nll_loss`，PyTorch 把它合成一个算子是为了数值稳定（log-sum-exp trick）。直接 `log(softmax(x))` 大 logit 会上溢。

## 3.4 信息论：所有 loss 的家谱

> **速记三条**：
> 1. **熵 = 不确定度**。LLM 给一个 token 99% 概率时熵很低（很确定），给一堆词均匀概率时熵很高（瞎猜）。`perplexity = exp(熵)` 是评测语言模型的标准指标。
> 2. **交叉熵 loss = 让模型给"正确答案"的概率尽量大**——和 3.3.5 的 MLE 是同一回事换了个名字。这就是预训练 / 微调的目标。
> 3. **KL 散度 = 两个概率分布的"差距"**——不对称（注意 $D_{KL}(p\|q) \neq D_{KL}(q\|p)$）。RLHF 用它防止模型"练废"、知识蒸馏用它让学生模仿老师。
>
> 3.4.4（log-sum-exp）记住"不要自己写 softmax+log，用 `F.cross_entropy`"就够了；3.4.5 互信息可以跳。
>
> → A 档读到这里可停（速记三条 + 知道 `F.cross_entropy` 在干嘛就够）；B 档继续看 3.4.1-3.4.4，3.4.5 互信息可跳；C 档把交叉熵 = MLE 的等价关系自己推一遍。

### 3.4.1 熵

离散分布 $p$ 的熵：

$$H(p) = -\sum_i p_i \log p_i$$

直觉：熵是这个分布的平均"不确定度"或"编码所需比特数"（log 取 2 时单位是比特）。均匀分布熵最大，确定性分布（某个 $p_i = 1$）熵为 0。

LLM 推理时下一个 token 的预测分布 entropy 越高代表模型越不确定。"perplexity" = $\exp(H)$，是评估语言模型的标配指标。

### 3.4.2 交叉熵

两个分布 $p, q$ 的交叉熵：

$$H(p, q) = -\sum_i p_i \log q_i$$

直觉：用分布 $q$ 的"码本"去编码服从 $p$ 的样本，平均要花多少 bit。$q$ 越接近 $p$，$H(p, q)$ 越小；$q = p$ 时最小，等于 $H(p)$。

分类任务里 $p$ 是 one-hot 的真实标签（只有真实类是 1），$q$ 是模型的 softmax 输出。代入公式：

$$H(p, q) = -\log q_{y_{\text{true}}}$$

也就是"模型给真实类的 log 概率取负"。这正是上面 NLL。**分类问题的交叉熵 loss 等于 MLE 的负对数似然**——同一个东西换了个说法。

（代码对应：`F.cross_entropy(logits, target)` 一行搞定，内部等价于 `F.nll_loss(F.log_softmax(logits, dim=-1), target)`。）

### 3.4.3 KL 散度

$$D_{KL}(p \| q) = \sum_i p_i \log \frac{p_i}{q_i} = H(p, q) - H(p)$$

符号读法：$D_{KL}(p \| q)$ 念作"$p$ 对 $q$ 的 KL 散度"，$\|$ 只是分隔符不是"除"。

直觉：**把 KL 散度当作"两个概率分布之间的差距"**——$p$ 和 $q$ 越像，KL 越小；完全相同时为 0。类比"信息浪费"：本来按 $p$ 编码最省 bit，但你拿 $q$ 的码本去编码服从 $p$ 的样本，多浪费的 bit 数就是 KL 散度。

性质：

- 非负：$D_{KL}(p \| q) \geq 0$，等号当且仅当 $p = q$。
- **不对称**：$D_{KL}(p \| q) \neq D_{KL}(q \| p)$。所以严格说不是"距离"。
- 当 $p$ 是固定的真实分布时，最小化 $D_{KL}(p \| q)$ 等价于最小化 $H(p, q)$（因 $H(p)$ 是常数）。

```python
import numpy as np

p = np.array([0.7, 0.2, 0.1])
q = np.array([0.5, 0.3, 0.2])

H_p = -np.sum(p * np.log(p))                    # 0.802
H_pq = -np.sum(p * np.log(q))                   # 0.971
KL = np.sum(p * np.log(p / q))                  # 0.169
print(np.isclose(KL, H_pq - H_p))               # True
```

**对应到 LLM 哪儿**：

1. 预训练 / 微调的 loss 是交叉熵，等价于让模型分布 $q_\theta$ 逼近数据分布 $p$。
2. RLHF（PPO 阶段）的目标是 $\mathbb{E}[r(x, y)] - \beta \cdot D_{KL}(\pi_\theta \| \pi_{\text{ref}})$。第二项约束 RL 训出来的策略 $\pi_\theta$ 别离参考模型 $\pi_{\text{ref}}$ 太远，避免"reward hacking 把模型说话能力练废"。$\beta$ 调大模型更保守、调小更激进。
3. 知识蒸馏：让学生模型的输出分布逼近教师模型，用的是 $D_{KL}(\text{teacher} \| \text{student})$ 或者反向版本，配合温度软化 logits。

### 3.4.4 数值稳定：log-sum-exp 与为什么不能裸算 softmax

为什么 PyTorch 把 `softmax + log + nll` 合成 `cross_entropy`？因为单独算会爆。

直接算 $\log\left(\frac{\exp(z_i)}{\sum_j \exp(z_j)}\right)$ 时，如果某个 $z_j$ 很大（比如 1000），$\exp(1000)$ 立即上溢成 `inf`，再除什么都得 NaN。即使在 fp32 下，最大正常浮点也就 $10^{38}$ 量级。

log-sum-exp trick：

$$\log \sum_j \exp(z_j) = c + \log \sum_j \exp(z_j - c)$$

取 $c = \max_j z_j$，让所有指数项都 $\leq 1$，安全。然后：

$$\log p_i = z_i - \log \sum_j \exp(z_j) = z_i - c - \log \sum_j \exp(z_j - c)$$

```python
import numpy as np

def log_softmax_safe(z):
    c = z.max(axis=-1, keepdims=True)
    return z - c - np.log(np.exp(z - c).sum(axis=-1, keepdims=True))

z = np.array([1000.0, 1001.0, 999.0])
print(log_softmax_safe(z))   # [-1.4076  -0.4076  -2.4076]
# 直接 np.exp(z) / np.exp(z).sum() 会全是 nan
```

这一段是工程师必备的"翻译公式时的潜规则"。看到论文里写 softmax，代码里实现是 log-sum-exp，不要奇怪。Attention 里的 softmax 同理——FlashAttention 的 online softmax 把这套技巧推到极致，在 SRAM 里逐块算 max 和 sum，避免实例化整个 $T \times T$ 注意力矩阵。

### 3.4.5 互信息

$$I(X; Y) = D_{KL}(p(x, y) \| p(x) p(y))$$

衡量"知道 $Y$ 后对 $X$ 的不确定性减少了多少"。$X, Y$ 独立时为 0，强相关时大。在表征学习（contrastive learning、CLIP）里隐含着最大化样本与正样本对的互信息。这部分细节留到 representation learning 章节。

## 3.5 优化：让模型从随机权重走到能用

> **速记三条**：
> 1. **训练 = 反复"算梯度、走一小步"**。这一小步多大叫学习率（lr），太大震荡、太小爬不动。
> 2. **AdamW 是 LLM 训练的默认优化器**，超参不用调，照搬 `lr=3e-4, betas=(0.9, 0.95), weight_decay=0.1`。理解为什么默认是这套就够了——AdamW 给每个参数自适应地缩学习率，比手调 SGD 省心。
> 3. **学习率不该全程不变**：先 warmup（从 0 缓慢升到 lr_max）、再 cosine decay（缓慢降到 lr_max 的 10%）。GPT、Llama 都这样。
>
> 3.5.5（梯度爆炸/消失的几何根源）"知道残差连接、LayerNorm 解决了这事"就行；3.5.7 可视化跳过不影响。
>
> → A 档读到这里可停（记住 AdamW + warmup + cosine 就够照搬）；B 档继续看 3.5.1-3.5.4 + 3.5.6 梯度裁剪，3.5.5 / 3.5.7 可跳；C 档把 Adam 的偏置修正和 AdamW 解耦自己推一遍。

### 3.5.1 SGD 与 Mini-batch

朴素梯度下降：

$$\theta_{t+1} = \theta_t - \eta \nabla_\theta \mathcal{L}(\theta_t)$$

（代码对应：`loss.backward()` 算出 $\nabla_\theta \mathcal{L}$ 存进 `param.grad`，`optimizer.step()` 执行 $\theta \leftarrow \theta - \eta \cdot \text{param.grad}$。）

$\eta$ 是学习率。每步用全数据算梯度太慢，于是用 mini-batch 估计：从训练集随机采 $B$ 条样本算平均梯度。这是随机梯度下降（SGD）。

batch size 是工程权衡：大 batch 梯度噪声小但泛化常常下降，小 batch 噪声大但带来某种隐式正则。LLM 预训练 batch 动辄上百万 token，靠 gradient accumulation + 数据并行拼出来。

### 3.5.2 Momentum：给下降一点惯性

纯 SGD 在狭长山谷里来回震荡。Momentum 维护一个速度向量：

$$\mathbf{v}_{t+1} = \mu \mathbf{v}_t + \nabla_\theta \mathcal{L}(\theta_t)$$
$$\theta_{t+1} = \theta_t - \eta \mathbf{v}_{t+1}$$

$\mu$ 一般取 0.9。直觉：球从山坡滚下来不是每帧只看当前坡度，还有惯性。震荡方向被相邻步抵消，主下降方向累加，收敛更快。

### 3.5.3 Adam 与 AdamW

直觉先行：Adam 比 SGD 多记两件事——**梯度的"近期平均方向"**（一阶矩，类似 momentum）和**梯度的"近期波动幅度"**（二阶矩，梯度平方的滑动平均）。然后用这两个一起决定每个参数实际走多大步：长期梯度大的参数自动减步、长期梯度小的自动加步。这就是"自适应学习率"的字面意思——每个参数有自己的有效 lr。

Adam 同时维护一阶矩（梯度均值）和二阶矩（梯度平方均值）：

$$\mathbf{m}_t = \beta_1 \mathbf{m}_{t-1} + (1 - \beta_1) \mathbf{g}_t$$
$$\mathbf{v}_t = \beta_2 \mathbf{v}_{t-1} + (1 - \beta_2) \mathbf{g}_t^2$$

$\mathbf{g}_t$ 是当前梯度。$\mathbf{m}_t$ 像 momentum；$\mathbf{v}_t$ 是梯度平方的滑动平均，可看作对角 Hessian 近似。

初始时 $\mathbf{m}_0 = \mathbf{v}_0 = 0$ 会让早期估计偏小，做偏置修正：

$$\hat{\mathbf{m}}_t = \mathbf{m}_t / (1 - \beta_1^t), \quad \hat{\mathbf{v}}_t = \mathbf{v}_t / (1 - \beta_2^t)$$

更新：

$$\theta_{t+1} = \theta_t - \eta \frac{\hat{\mathbf{m}}_t}{\sqrt{\hat{\mathbf{v}}_t} + \epsilon}$$

每个参数的有效学习率是 $\eta / (\sqrt{\hat{v}} + \epsilon)$，对应上面"自适应"那句直觉。默认超参 $\beta_1 = 0.9, \beta_2 = 0.999, \epsilon = 10^{-8}$，绝大多数论文不调这三个。

**AdamW** 的关键改动：把 weight decay 从 loss 里抽出来，直接打在参数上：

$$\theta_{t+1} = \theta_t - \eta \left(\frac{\hat{\mathbf{m}}_t}{\sqrt{\hat{\mathbf{v}}_t} + \epsilon} + \lambda \theta_t\right)$$

为什么要解耦？如果把 L2 正则 $\frac{\lambda}{2}\|\theta\|^2$ 加到 loss 里，它的梯度 $\lambda \theta$ 会被 Adam 的自适应学习率缩放——梯度大的参数受到的正则反而更弱，违反了 L2 想要"对所有参数施加一致约束"的初衷。AdamW 跳过这个缩放直接乘上 $\lambda \theta$。从 BERT、GPT 到 Llama，预训练 LLM 几乎全用 AdamW，weight decay 一般 0.1。

```python
import torch.optim as optim

# 实际工程里你只需要这一行
optimizer = optim.AdamW(model.parameters(), lr=3e-4, betas=(0.9, 0.95),
                        weight_decay=0.1)
```

注意 LLM 训练里 $\beta_2$ 常调到 0.95，因为 $\beta_2 = 0.999$ 在大数据小 step 下二阶矩还没"暖好"。

### 3.5.4 学习率调度

学习率不该全程不变。常用调度：

- Linear warmup：前 $T_w$ 步学习率从 0 线性增到 $\eta_{\max}$。冷启动避免 Adam 二阶矩没估稳就大步走。
- Cosine decay：warmup 之后从 $\eta_{\max}$ 沿余弦曲线降到 $\eta_{\min}$（常取 $0.1 \eta_{\max}$）。

```python
import math
def lr_at_step(step, warmup, total, lr_max, lr_min):
    if step < warmup:
        return lr_max * step / warmup
    progress = (step - warmup) / (total - warmup)
    return lr_min + 0.5 * (lr_max - lr_min) * (1 + math.cos(math.pi * progress))
```

GPT-3、Llama 系列基本都是 warmup + cosine。

### 3.5.5 梯度爆炸 / 消失的几何根源

为什么深网容易梯度爆炸或消失？反向传播是矩阵连乘——回想 3.2.2 的链式法则。一个 $L$ 层网络，loss 对第 1 层的梯度是

$$\frac{\partial \mathcal{L}}{\partial \theta_1} = \frac{\partial \mathcal{L}}{\partial h_L} \prod_{l=2}^{L} \frac{\partial h_l}{\partial h_{l-1}} \cdot \frac{\partial h_1}{\partial \theta_1}$$

中间一长串 Jacobian 矩阵相乘。如果这些 Jacobian 的奇异值平均大于 1，乘起来指数爆炸；平均小于 1，指数衰减到 0。这就是为什么权重初始化要让"层间方差守恒"（Xavier、Kaiming），为什么需要残差连接（直接给梯度一条不经过 Jacobian 的高速路），为什么 LayerNorm / RMSNorm 重要（控制激活范数）。

ReLU 的导数是 0 或 1，比 sigmoid（导数最大 0.25）更不容易梯度消失，这是它能撑起深网的几何原因之一。Transformer 把残差连接、LayerNorm、ReLU/GELU 全用上，就是让这条 Jacobian 链在数百层下仍然可训练。

### 3.5.6 梯度裁剪

训练 RNN / Transformer 偶尔会遇到梯度爆炸（loss 突然 NaN）。梯度裁剪给整个梯度向量的范数设上限：

$$\mathbf{g} \leftarrow \mathbf{g} \cdot \min\left(1, \frac{c}{\|\mathbf{g}\|}\right)$$

PyTorch 一行：`torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)`。LLM 预训练里 $c = 1.0$ 是默认；超过这个值整体缩回 1，方向不变只缩长度。

## 3.6 几何直觉：为什么高维空间反直觉

> **速记三条**（这节是 RAG 工程师的命门，不要跳）：
> 1. **Embedding = 把一段文本/图像变成一个向量**，语义相近的向量距离近。RAG 检索的全部魔法就在这里。
> 2. **不同模型的 embedding 不可互换**。`text-embedding-3-small` 和 `bge-large-zh` 的相似度数字没有可比性，换模型就要重建索引。
> 3. **cosine 相似度不是真的"语义距离"，只是工程界默认的代理**。它在某些情况下会失效（跨语言、跨领域、各向异性），所以 RAG 工程里检索后必须做 reranker（第二段重排）。
>
> 3.6.2 维度诅咒读一遍有印象就行——它解释了"为什么需要 reranker"，跳过不影响调通 RAG。

### 3.6.1 Embedding 空间

把每个 token / 句子 / 图像映射到一个 $d$ 维向量，使得"语义相近的项"距离近，这个映射叫 embedding。Word2Vec、GloVe、BERT 的 [CLS]、sentence-transformers 都是干这件事。

embedding 空间的几何性质决定了下游能做什么：

- 相似度检索：cosine 或点积。RAG 系统的核心就是把 query 和 doc 都 embed 然后做 KNN。
- 类比：经典例子 "king - man + woman ≈ queen"。在 word2vec 时代很 work，现代 contextual embedding 上效果反而没那么干净，因为同一个词在不同 context 下向量不同。
- 聚类：embedding 上跑 KMeans / HDBSCAN 能做无监督主题归纳。

### 3.6.2 维度诅咒

高维空间几个反直觉的事实，记住能避开调参坑：

**1. 单位球体积集中在表面附近。** $d$ 维球壳 $r \in [1-\epsilon, 1]$ 的体积占比是 $1 - (1-\epsilon)^d$，$d = 100, \epsilon = 0.01$ 时是 $63\%$；$d = 1000$ 时几乎 $100\%$。意思是高维球里的"内容"几乎全在皮上。

**2. 任意两个随机向量近似正交。** 在 $d$ 维中，两个独立 standard normal 向量的 cosine 相似度均值 0、标准差 $1/\sqrt{d}$。$d = 1000$ 时几乎所有随机向量两两正交。这有好处（embedding 容量近似指数级）也有坏处（朴素的"距离阈值过滤"在高维里很容易失效）。

**3. "最近邻"和"最远邻"在高维差距很小。** 数据足够分散时，对一个 query 距离最远的点和最近的点，距离比值会随维度上升趋近 1。这意味着高维 KNN 在没结构的数据上会失效。RAG 之所以还能工作，是因为 embedding 空间不是均匀填满的，语义相关的内容形成低维流形。

```python
import numpy as np

for d in [10, 100, 1000, 10000]:
    a = np.random.randn(2000, d)
    a /= np.linalg.norm(a, axis=1, keepdims=True)
    sim = a[:1000] @ a[1000:].T
    print(f'd={d}: cos mean={sim.mean():.4f}, std={sim.std():.4f}')
```

输出会发现 std 大致 $1/\sqrt{d}$，跟理论一致。

### 3.6.3 Cosine 为什么常用

回到 3.1.2 的悬念。语义相似度任务里 cosine 比欧氏距离更稳，原因：

- embedding 的范数常常和"信息量 / 频率"相关，不一定和"语义"相关。一个高频词的向量长度可能很大，但你不希望它和所有东西看起来都"远"。归一化后只看方向。
- 现代 embedding 模型（OpenAI、BGE、E5）训练时损失就是 contrastive cosine，输出已对齐到单位球。
- 高维空间欧氏距离对范数敏感，在未归一化向量上很容易被 outlier 主导。

工程口径：用 sentence-transformers 系统直接点积；用未归一化向量先归一化再点积；不要混用 metric。

**但 cosine 不是免费午餐**——LLM 的 contextual embedding 训练后并不真的「均匀分布在单位球面上」，这个现象叫做 anisotropy（各向异性）：所有向量被压缩进单位球的一个狭窄锥形里，导致随机两个无关句子的 cosine 也能高到 0.6 以上。Gao et al. 2019 "Representation Degeneration" 最早系统讨论；2024 NVIDIA "Is Cosine-Similarity of Embeddings Really About Similarity?" 指出 cosine 在某些线性模型 embedding 上甚至和"语义相似性"没有可靠对应——它只是工程界默认接受的代理。

实际后果与对策：

- 不要用绝对 cosine 阈值（"> 0.85 就算相似"）跨数据集迁移。同一阈值在 BGE 和 OpenAI embedding 上含义完全不同，必须按数据集校准。
- 检索质量差时先看分布：把 query-doc 的 cosine 直方图画出来，如果众数已经 > 0.7、长尾压缩在 0.05 区间内，说明 anisotropy 严重，可分性差。
- 缓解：用 BGE / E5 / GTE 这种做过 contrastive 后训练的模型（几何性质比 word2vec / 早期 BERT 好得多），或者 whitening / centering（减去全局均值再归一化）把分布拉回各向同性。
- 高维下还有"hubness"：少数向量成为大量 query 的近邻，污染 top-K 召回。BGE-reranker 这类 cross-encoder 重排就是为了过滤这部分污染，RAG 工程里 retrieval + rerank 两段式是必需。

一句话：cosine 不是"真"的语义距离，是模型训练目标顺手定义出来的几何代理。失效时（跨语言、跨领域、长文档片段）先怀疑几何而不是模型。

## 3.7 采样与生成：温度、Top-K、Top-P 的数学

> **速记三条**（调 OpenAI / Claude API 的工程师必读）：
> 1. **`temperature` 是 logits 的"对比度旋钮"**：`T=0` 等价 argmax 贪婪解码（每次必出最高概率词，确定 / 死板）；`T=1` 是原版；`T>1` 输出更随机更有创意。
> 2. **`top_p=0.9`（nucleus 采样）= "只在概率累加到 90% 的那批 token 里挑"**——比 top_k 自适应、是商业 API 的事实标准。
> 3. **不同任务用不同参数**：见 3.7.5 末尾的表。**`temperature` 和 `top_p` 通常只调一个**，另一个保持默认。
>
> 3.7.4 重复惩罚记住有这玩意儿就行，调起来时翻回来看。

### 3.7.1 从概率分布到一个 token

LLM 推理流程：模型 forward → 拿到最后一个位置的 logits（形状 $V$）→ 应用采样策略 → 选 1 个 token id → append 到序列 → 再 forward。这是个循环。每一步是从一个 $V$ 维 categorical 分布里抽 1 个样本，所有"温度 / top-k / top-p"都在调整这个分布的形状再抽样。

### 3.7.2 Softmax 与温度

模型的最后一层输出 logits $\mathbf{z} \in \mathbb{R}^V$，$V$ 是词表大小。softmax 把 logits 变成概率：

$$p_i = \frac{\exp(z_i)}{\sum_{j=1}^V \exp(z_j)}$$

加温度 $T$：

$$p_i(T) = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)}$$

（代码对应：`F.softmax(logits / T, dim=-1)`；`T=0` 在 API 层一般等价于 `torch.argmax(logits, dim=-1)`，因为 $T \to 0$ 时 softmax 退化为 one-hot。）

- $T \to 0$：分布变 one-hot，等价贪婪解码（argmax）。
- $T = 1$：原始 softmax。
- $T \to \infty$：分布趋向均匀，完全随机。

直觉：温度是 logits 的"对比度"调节钮。$T < 1$ 把高 logit 拉得更高、低的压得更低，输出更确定也更死板。$T > 1$ 把所有 logit 拉平，更多样也更容易胡说。

```python
import numpy as np

logits = np.array([2.0, 1.0, 0.1, -0.5])
for T in [0.5, 1.0, 2.0]:
    p = np.exp(logits / T)
    p /= p.sum()
    print(f'T={T}: {np.round(p, 3)}')
```

### 3.7.3 Top-K 与 Top-P (Nucleus)

直接从全词表 $\text{Cat}(\mathbf{p})$ 采样会有"长尾问题"——词表 5 万词，每个词概率 $10^{-6}$ 加起来也能踩中一个，于是输出乱码。

**Top-K**：只保留概率最高的 $K$ 个 token，重新归一化后采样。$K$ 常取 50。简单粗暴，问题是模型很自信时（top-1 概率 0.95）剩下 49 个低质量噪声还在被采。

**Top-P / Nucleus**：把概率从大到小排序，累加直到超过阈值 $p$（如 0.9），只在这部分里采样。集合大小自适应——模型自信时集合可能就 1-2 个 token；不确定时可能展开到几十个。这是目前 LLM 默认采样策略。

```python
import torch
import torch.nn.functional as F

def top_p_sample(logits, p=0.9, T=1.0):
    """1-D logits 版本。多 batch 需要把 sorted_idx 用 gather(-1, pick) 取回。"""
    probs = F.softmax(logits / T, dim=-1)
    sorted_probs, sorted_idx = probs.sort(descending=True)
    cumsum = sorted_probs.cumsum(dim=-1)
    # cumsum - sorted_probs 是「不含当前位置」的前缀和，超过 p 的位置说明
    # 之前已经凑够 p、当前 token 严格在阈值之后，整个砍掉。
    # 这是 HuggingFace 的标准做法：保留第一个让累加跨过 p 的 token。
    mask = cumsum - sorted_probs > p
    sorted_probs = sorted_probs.masked_fill(mask, 0.0)
    sorted_probs = sorted_probs / sorted_probs.sum()
    pick = torch.multinomial(sorted_probs, 1)
    return sorted_idx[pick]
```

参数搭配见 3.7.5 表格。OpenAI / Anthropic API 的 `temperature` 和 `top_p` 通常建议只调一个、另一个保持默认。

**生产里实际用什么**（截至 2026 年 5 月）：

- 商业 API（OpenAI / Anthropic / Gemini）只暴露 `temperature` 和 `top_p`。`top_k` 这些年逐渐退出（Anthropic 2025 年仍有但默认不开），"Top-K vs Top-P 之争"在调 API 工程师层面已是历史问题。
- min-p（Minh et al. 2024）用相对阈值——保留概率不低于 `min_p × p_max` 的 token，自适应跟随分布锐度。开源生态（vLLM、llama.cpp、SGLang）都支持，主流商业 API 不暴露。跑 vLLM 做创意类生成可以试 `min_p=0.05, temperature=1.0`。

第 14 章会把采样参数和 prompt 工程一起重新讨论：生产里温度选择往往不是数学问题而是工程取舍——结构化输出固定 `T=0` 才能让 prompt caching 命中率稳定、A/B 测试可重复。

### 3.7.4 重复惩罚与 logit bias

Top-K / Top-P 解决"乱采到长尾"，但解决不了"一直采到同一个高概率词"。常见技巧：

- **frequency penalty / presence penalty** （OpenAI 风格）：对已经出现过的 token，logit 减去一个惩罚值，下一步采样时它就更难出来。frequency 是按出现次数累加，presence 是只要出现过就扣一次。
- **repetition penalty**（HuggingFace 风格）：对已出现 token 把 logit 除以 $r > 1$（正 logit 变小、负 logit 变大）。和 OpenAI 风格语义一致但实现细节不同。
- **logit bias**：直接给特定 token id 的 logit 加常数。强制 / 禁止某些 token 的硬手段。

```python
def apply_freq_penalty(logits, history, alpha=0.5):
    counts = torch.bincount(history, minlength=logits.size(-1))
    return logits - alpha * counts.float()
```

### 3.7.5 贪婪 vs 采样

贪婪解码每步取 argmax：确定性、可复现，但容易陷入循环 ("the the the the")。这是因为模型一旦给某 token 略高概率，下一步同样的 context 还会推它，正反馈。采样打破这种循环。

代价是不可复现，除非把 RNG seed 也固定。生产里你会遇到的取舍：

| 任务 | 推荐 |
| --- | --- |
| 提取 JSON / 工具调用 | `temperature=0`（贪婪） |
| 翻译 / 摘要 | `temperature=0.3-0.5` |
| 闲聊 / 创作 | `temperature=0.7-1.0, top_p=0.9` |
| 代码生成 | `temperature=0.2`（OpenAI）或 `temperature=0`（要求严格） |

## 3.8 综合实战：从零写一个 self-attention

> **速记三条**：
> 1. **self-attention 的核心一行公式 $\text{softmax}(QK^T/\sqrt{d_k})V$ 在做的事**：先算"每个 token 该看其他哪些 token"（$QK^T$，输出一张注意力权重表），再用这张权重表把 value 加权平均。
> 2. **生产代码不要手写**——`torch.nn.functional.scaled_dot_product_attention(Q, K, V, is_causal=True)` 一行就好，PyTorch 会自动跑 FlashAttention 后端。
> 3. **手写一遍的唯一目的是 debug 时能在脑子里拆开每一步**。如果你不打算改造 attention（加 LoRA、改 mask 形状、调位置编码），完全可以跳过这一节直接看 3.9 速查表。

### 为什么 2026 年还要自己写一遍

FlashAttention-3、PagedAttention、`F.scaled_dot_product_attention` 已经把高性能 attention 包成一行 API；生产里你不会 naive 地实例化 $T \times T$ 注意力矩阵（8K 上下文就是 64M 个 float32，长上下文直接 OOM）。所以"自己写一遍"听起来像数学家的浪漫。但仍然让你写一次，按重要性排序：

1. **调试需要**。训练 loss 不下降、attention 分布很怪、mask 形状对不上时，你需要在脑子里对着公式拆开 forward 一步步看哪里坏。FlashAttention 是黑盒，里面没法插 `print`。
2. **改造需要**。RoPE、ALiBi、attention sink、sliding window mask、把 LoRA 注入 $W_q, W_v$——所有这些都不在 FlashAttention 接口里，要在自己的实现层面改。
3. **教学顺序需要**。前面所有概念（matmul、softmax、log-sum-exp、链式求导、广播）在这里第一次合在一起跑。

下面这段是"读一次留个心智模型，生产代码换成内置实现"。

---

把前面所有概念串起来。Transformer 的核心一行公式：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right) \mathbf{V}$$

逐项拆解：

- $\mathbf{Q}, \mathbf{K}, \mathbf{V}$ 都是输入 embedding 经过三个线性层得到的。形状 $(\text{seq}, d_k)$，$d_k$ 是每头维度。
- $\mathbf{Q}\mathbf{K}^T$ 是 $(\text{seq}, \text{seq})$ 的 score 矩阵，$(i, j)$ 元素是 query $i$ 和 key $j$ 的点积——衡量"位置 $i$ 该多关注位置 $j$"。
- 除以 $\sqrt{d_k}$ 是缩放，防止 $d_k$ 大时点积方差大、softmax 进入饱和区。
- softmax 在最后一维做，把每行变成概率（加权系数）。
- 乘 $\mathbf{V}$ 把每个位置的 value 按权重相加，得到该位置的新表示。

整段代码：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class SingleHeadAttention(nn.Module):
    def __init__(self, d_model, d_k):
        super().__init__()
        self.d_k = d_k
        self.W_q = nn.Linear(d_model, d_k, bias=False)
        self.W_k = nn.Linear(d_model, d_k, bias=False)
        self.W_v = nn.Linear(d_model, d_k, bias=False)

    def forward(self, x, mask=None):
        # x: (B, T, d_model)
        Q = self.W_q(x)              # (B, T, d_k)
        K = self.W_k(x)              # (B, T, d_k)
        V = self.W_v(x)              # (B, T, d_k)

        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)   # (B, T, T)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(scores, dim=-1)                          # (B, T, T)
        out = attn @ V                                            # (B, T, d_k)
        return out, attn


class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.n_heads = n_heads
        self.d_k = d_model // n_heads
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, T, _ = x.shape
        # split into heads: (B, T, d_model) -> (B, n_heads, T, d_k)
        Q = self.W_q(x).view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, T, self.n_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, T, self.n_heads, self.d_k).transpose(1, 2)

        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)    # (B, h, T, T)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(scores, dim=-1)
        out = attn @ V                                            # (B, h, T, d_k)
        out = out.transpose(1, 2).contiguous().view(B, T, -1)
        return self.W_o(out)


# 因果 mask：位置 i 只能看到 0..i。返回 (T, T)，靠广播适配 (B, T, T) 或 (B, h, T, T)
def causal_mask(T):
    return torch.tril(torch.ones(T, T))


B, T, d_model, n_heads = 2, 8, 64, 4
x = torch.randn(B, T, d_model)
mha = MultiHeadAttention(d_model, n_heads)
y = mha(x, mask=causal_mask(T))
print(y.shape)            # torch.Size([2, 8, 64])
```

跑一遍，观察：

1. `Q @ K.transpose(-2, -1)` 是 batched matmul，对应 3.1.3。
2. `/ math.sqrt(self.d_k)` 是缩放，跟正态分布的方差控制有关——当 $\mathbf{q}, \mathbf{k}$ 各分量独立、方差 1 时，点积方差是 $d_k$，除以 $\sqrt{d_k}$ 让方差归 1。
3. `masked_fill(... -inf)` 是工程技巧：softmax 之前置 $-\infty$，softmax 后变 0。因果 mask 让位置 $i$ 看不到未来，这是 GPT 类 decoder-only 模型的核心。
4. `F.softmax(scores, dim=-1)` 在 key 维度归一化，结果每行概率和为 1。
5. `attn @ V` 是按权重的 value 加权和。

整个模块不到 50 行。把 GPT-2 拆开来看，你会发现它就是「embedding + 12 个这样的 attention block + 一个 LM head」的堆叠。一切复杂度都建立在这一段矩阵运算上。

### 生产里你应该调什么

这段 50 行在两处让生产爆炸：`Q @ K.transpose(-2, -1)` 实例化了 $T \times T$ 的 score 矩阵，长上下文直接 OOM；`softmax(scores) @ V` 又一次访问全矩阵，内存带宽是瓶颈。FlashAttention（Dao et al. 2022 / 2023 / 2024 三代）通过分块 + online softmax 在 SRAM 里逐块算掉，从不实例化完整 score 矩阵，端到端做到 $O(N)$ 内存、$2$-$4\times$ 加速。生产代码、HuggingFace transformers、vLLM、SGLang 全走 `F.scaled_dot_product_attention(Q, K, V, is_causal=True)` 这条路径，PyTorch 2.x 会自动选 FlashAttention / Memory-Efficient / Math 后端。

### 一个练习：从零写 word2vec skip-gram

这个练习把 embedding、cross-entropy、SGD 全用上。给定一句话，让中心词预测上下文词。

```python
import torch
import torch.nn as nn

corpus = "the quick brown fox jumps over the lazy dog the fox is quick".split()
vocab = list(set(corpus))
w2i = {w: i for i, w in enumerate(vocab)}
V = len(vocab)
embed_dim = 8

# 生成 (center, context) 对，window=2
pairs = []
for i, w in enumerate(corpus):
    for j in range(max(0, i-2), min(len(corpus), i+3)):
        if j != i:
            pairs.append((w2i[w], w2i[corpus[j]]))

class SkipGram(nn.Module):
    def __init__(self, V, d):
        super().__init__()
        self.in_emb = nn.Embedding(V, d)
        self.out_emb = nn.Embedding(V, d)
    def forward(self, center):
        v_c = self.in_emb(center)        # (B, d)
        # 简化版：和全词表点积，cross-entropy 把这个 (B, V) 当作每行一个分布
        logits = v_c @ self.out_emb.weight.T   # (B, V)
        return logits

model = SkipGram(V, embed_dim)
opt = torch.optim.Adam(model.parameters(), lr=0.05)
centers = torch.tensor([p[0] for p in pairs])
contexts = torch.tensor([p[1] for p in pairs])

for epoch in range(200):
    logits = model(centers)
    loss = nn.functional.cross_entropy(logits, contexts)
    opt.zero_grad(); loss.backward(); opt.step()

# 看看 fox 和 dog 的向量近不近
import torch.nn.functional as F
f = model.in_emb.weight[w2i['fox']]
d = model.in_emb.weight[w2i['dog']]
print('cos(fox, dog) =', F.cosine_similarity(f, d, dim=0).item())
```

跑完会看到 fox 和 dog 的 cosine 相似度是个不为 0 的值（在这个 toy 语料里它们都和 quick / the 共现）。这就是 word2vec 的核心：通过预测上下文，词的语义关系编码进 embedding 几何。

实战里 word2vec 还要加 negative sampling（不和全词表算 softmax，只采 5-20 个负样本算 sigmoid loss）来加速；本节的简化版在大词表下慢但思想完整。

## 3.9 用数学直觉调模型：常见问题对应到哪一节

把这章的内容和实际工程症状挂钩，是真正能记住的方式。下面这张表请收藏：

| 现象 | 大概率根源 | 对应章节 |
| --- | --- | --- |
| Loss 第一步就是 NaN | logits 太大、softmax 上溢；或者除以 0 | 3.4.4 数值稳定 |
| Loss 几个 step 后突然飙升 / NaN | 梯度爆炸 | 3.5.5 / 3.5.6 |
| Loss 缓慢下降然后停滞 | learning rate 太低、或陷入鞍点 | 3.5.1 / 3.5.4 |
| Loss 震荡不下降 | learning rate 太高 / batch 太小 | 3.5.1 / 3.5.4 |
| 训练 loss 下降、验证 loss 上升 | 过拟合，需要更多正则 / 数据 | 3.3.4（先验=正则） |
| RAG 检索召回质量差 | embedding 没归一化或 metric 选错 | 3.1.2 / 3.6.3 |
| LLM 输出复读机 | 贪婪解码 + 模型自信度过高 | 3.7.5 |
| LLM 输出一会儿胡说 | 温度太高 / top-p 太大 | 3.7.2 / 3.7.3 |
| RLHF 训出来模型话都不会说了 | KL 约束系数 $\beta$ 太小 | 3.4.3 |
| 微调后小模型遗忘了原能力 | 学习率过大 / 没有 KL / 没有 L2 | 3.4.3 / 3.5.3 |
