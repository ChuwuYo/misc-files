# 决策记录：i18n 多语言支持 — 规划 (v2)

日期：2026-05-17（v1）/ 2026-05-17 修订 v2（吸收对抗审查 5 MUST + SHOULD）
状态：**规划已定，待 P0 验证后实施**

## 1. 背景

项目当前仅 zh-CN 硬编码 UI。需求：上多语言覆盖海外华人/留学生与国际用户。
本文档锁定所有规划决策，作为后续分阶段实施的契约。

## 2. 已锁决策

### 2.1 库选型 — **Paraglide JS** (@inlang/paraglide-js v2.x)

- 编译期 tree-shake，每条消息编译为类型安全函数
- 初始运行时 <500B gzip，按用引；路由分包自动适配
- Inlang 社区生态活跃，Svelte5 通过 runes 包薄壳
- **备选 kill-switch**：若 P0 中 Paraglide 与"纯 Vite（非 SvelteKit）"集成
  存在阻塞（locale strategy 必须 cookie/URL 而非 localStorage），改用
  **@nanostores/i18n**（Svelte5 runes 原生、~1KB 运行时）。决策点在 P0 末

被否方案：svelte-i18n（19 月未更新）、@tolgee/svelte（依赖云平台与"纯本地"
理念冲突）、typesafe-i18n（维护轨迹不稳）、@lingui/core（无 Svelte 绑定）。

### 2.2 v1 语种集 — 5 个，**脚本制 BCP47** 命名

```
zh-Hans (默认) · zh-Hant · en · ja · ko
```

理由：脚本制（zh-Hans/zh-Hant）符合 BCP47 最佳实践；未来 SG/MY zh-Hans、
HK zh-Hant 可复用，不再独立增列。**所有文件/路径/键以脚本码表示，**
**不出现 `tw/jp/kr` 这类区域简写**。

### 2.3 翻译质量保证

- 我（AI）为 zh-Hant / ja / ko 生成起版，每文件末追加注释
  `<!-- pending native review: <yyyy-mm-dd> -->`
- 发布后追迹补译；不阻塞 v1 上线
- README 与发布说明须标注当前译文质量级别
- 后续若引入译者协作平台（Crowdin/Tolgee），从此 ADR 之后开新决策

### 2.4 品牌策略

**所有界面、所有语种保留 `方寸 Eikon` 双语标识不变**。CJK 字符在 en 系统
由 fallback 中文字体渲染（已验证可读）。

### 2.5 序号符号 `①②③④`

**保留不动**。Unicode 通用符号，各语种渲染一致；保留现有设计品质。

### 2.6 文件名 — 跟随界面语言

| Locale | 单张 | 六寸排版 | 图片工具压缩 |
|---|---|---|---|
| zh-Hans | `证件照-{name}-{w}x{h}.{ext}` | `证件照-六寸排版-{name}.jpg` | `image-tool.{ext}` (TBD) |
| zh-Hant | `證件照-{name}-{w}x{h}.{ext}` | `證件照-六寸排版-{name}.jpg` | 同上 |
| en | `id-photo-{name}-{w}x{h}.{ext}` | `id-photo-6inch-sheet-{name}.jpg` | 同上 |
| ja | `証明写真-{name}-{w}x{h}.{ext}` | `証明写真-6inch-{name}.jpg` | 同上 |
| ko | `증명사진-{name}-{w}x{h}.{ext}` | `증명사진-6inch-{name}.jpg` | 同上 |

**实现接口**（M1 修订）：

- `domain/sizes.ts` 给每条 `PhotoSpec` 加 `nameKey: string` + 可选 `noteKey?: string`
- 三处文件名生成点全部走 i18n 模板键（**枚举完整**）：
  1. `core/pipeline.ts:fileBase()` 单张照
  2. `core/pipeline.ts:46` 六寸排版（**当前硬编码字面量**，必须改键化）
  3. `components/tools/CompressPanel.svelte:58` `image-tool.{ext}`
- 新增 `filename_*` 键桶（§2.9）：`filename_id_photo` / `filename_id_photo_sheet` /

### 2.7 UI 切换器 — 顶栏 🌐 图标 + 下拉菜单

- 位置：TopNav 中 `ThemeToggle` 左侧
- 触发：32×32 圆形图标按钮，IconaMoon `iconamoon/language`（**N1 已决**，
  比 `world` 更语义化）
- 菜单：本地名展示
  - `简体中文` / `繁體中文` / `English` / `日本語` / `한국어`
  - 当前项右侧 ✓，`aria-current="true"`，行内 `lang="zh-Hans"` 等
- 行为：Esc 关、点外关、上下方向键、聚焦回退到触发按钮、首次焦点落在当前项
- 移动端（≤960px）：弹层右对齐，`max-width: calc(100vw - 16px)`

  `已切换至 简体中文` / `Switched to English` 等

### 2.8 命名策略

- 寸尺寸：双语 hybrid。en/ja/ko 统一 Pinyin 转写 `1-cun (25×35mm)`
  / `1cun (25×35mm)`（**详见 [terminology.md A 节](../i18n/terminology.md#a-photo-size-unit-names-寸-cluster--hybrid-policy-per-plan-§28)**）
- 证件文档：功能化翻译（详见 terminology.md B 节）
- 签证：直译（detail in C 节）
- 相纸：真英寸（`6-inch Paper (102×152mm)` — 与寸尺寸不同语义）
- 底色：`White / Blue / Red` + 长形 `… background`
- 自定义颜色 zh-Hant `自訂`（TW 习惯，非 PRC `自定义`）

### 2.9 架构原则

- 新 `src/lib/state/lang.svelte.ts`（镜像 theme.svelte.ts；try/catch 包裹
  localStorage；初始 `localStorage → navigator → zh-Hans` 链）
- `#apply()` 写两处：`document.documentElement.lang` + 触发标题重算
- **生成产物处理**（**S7 修订**）：`src/lib/paraglide/` **gitignored**，
  加 `prebuild` / `predev` 脚本生成。理由：避免每次消息编辑产生 commit
  噪声；svelte-check 不读陈旧生成代码
- 消息文件：`messages/zh-Hans.json` 等 5 个
- Inlang 项目：`project.inlang/settings.json`
- 键名：扁平 snake_case，按**界面意图**分桶：
  - `chrome_*`（顶栏/路由/品牌/语言切换）
  - `source_*`（上传/校验）
  - `maker_size_*`（尺寸库/搜索/自定义）
  - `maker_crop_*`（裁剪工具栏）
  - `maker_side_*`（背景/格式/导出）
  - `tools_adjust_*`（实时调整）
  - `tools_compress_*`（格式与下载）
  - `tools_preview_*`（预览查看器）
  - `common_*`（重置/取消/确认/分隔符 ≤15 键）
  - `domain_size_<id>_name` / `domain_size_<id>_note`（数据标签）
  - `domain_format_*`（格式墙错误）
  - `domain_error_*`（domain 抛错）
  - `core_error_*`（core 抛错）
  - `filename_*`（导出文件名模板）

- **保持 domain 纯**（**M3 修订**）：`domain/formats.ts:validateImageFile`
  返回 **`{ kind: 'error', key: string, params?: Record<string,unknown> } | null`**
  而非 string；UI 层调 `m[key](params)` 解析
- 共有 **13 处** `throw new Error(中文/英文)` 散落各处，迁移采用 **`AppError`**
  子类承载 `{key, params}`，捕获方在显示前解析。所有点位枚举见
  Phase 2 表
- **`Intl.NumberFormat` 包**：新建 `src/lib/domain/intl.ts`，
  `fmtBytes(n)` / `fmtPercent(n)` **保持原签名不带 locale 参数**，
  内部读 `lang` 单例，调用方零改动
- `PHOTO_SPECS` 加 `nameKey: string` + `noteKey?: string`；保留 `name` 字段
  作 zh-Hans 起源 fallback（仅 Phase 3 过渡期使用）

### 2.11 尺寸库区域过滤 UX（与 UI locale **解耦**）

**问题**：补全后 ~35+ 条目；UI 语种 ≠ 用户证件区域（在日华人读中文 UI
但要 JP 履歴書）。两轴正交：**区域**（哪国/地区）× **用途分类**（证件 / 签证 / 相纸 / …）。

**方案 A（用户已选）**：搜索框上方加 **多选区域 chip 行** + 现有分类手风琴。

```
[国际] [中国大陆] [台湾] [香港] [日本] [韩国]    [全选]
（默认按 UI locale 智能预选）

搜索框 [...]

▼ 常用    （按所选区域过滤后的条目数）
▼ 证件
▼ 护照签证
▼ 证书考试
▼ 相纸
```

**行为规则**：
- 多选：可同时显示 `中国大陆 + 日本`
- 默认预选（locale-derived 智能默认）：
  - `zh-Hans` → `[国际, 中国大陆]`
  - `zh-Hant` → `[国际, 台湾, 香港]`
  - `en` → `[国际]`
  - `ja` → `[国际, 日本]`
  - `ko` → `[国际, 韩国]`
- **用户首次操作 chip 后，停止跟随 locale 切换**（持久化 `eikon-regions`
  到 localStorage）；后续 UI 语种切换不重置区域选择
- 搜索时：**区域过滤让步**——搜索结果跨所有区域显示（避免「我知道名字却
  搜不到」），底栏弱提示 `搜索时显示全部区域`
- `[全选]` chip：单击切换全选/全不选所有具名区域；全选时与单独选 5 个等价

**实现细节**：
- 状态：新 `chrome.regions: Set<Region>` 局部状态于 SizeLibrary，或更广
  范围（如多页复用）放 `state/library.svelte.ts`
- 初始化：读 localStorage `eikon-regions` → 否则按 lang 推默认集
- 持久化：用户手动调整后即写
- `i18n` 键：
  - `chrome_region_intl` / `_cn` / `_tw` / `_hk` / `_jp` / `_kr` = 6 键 × 5
    语种 = 30 译条
  - `chrome_region_all` / `chrome_region_search_overrides` 各 1 × 5 = 10
- **每条 `PhotoSpec` 必填 `region`**：现有 23 条按表落 region
- 新条目（§2.10 P0 12 条）region 显然按 id 前缀（`jp-*` → `jp` 等）

**区域归属表**（现有 + P0 新增）：

| region | 当前条目（id） | P0 新增 |
|---|---|---|
| **cn** | cn-1cun, cn-1cun-s, cn-1cun-l, cn-2cun, cn-2cun-s, cn-2cun-l, cn-idcard, cn-social, cn-driver, cn-marriage, cn-passport, cn-ehk, cn-degree, cn-teacher, cn-ncre, cn-psc | — |
| **tw** | — | tw-id, tw-passport |
| **hk** | — | hk-passport, hk-id |
| **jp** | jp-visa | jp-rireki, jp-passport, jp-mynumber, jp-driver, jp-zairyu |
| **kr** | — | kr-passport, kr-id, kr-driver |
| **intl** | us-visa, schengen-visa, uk-visa, paper-5, paper-6 | — |

注：寸-cluster（一寸/小一寸…）归 **cn** 类。相纸（paper-5/6）归 **intl**
（5/6 英寸相纸全球通用）。`cn-ehk`（港澳通行证）归 **cn**（PRC 居民申办的
通行证件）。

**视觉**：chip 设计与现有 `.tab` pill 一致体系；选中态 accent 填充；
未选用 `--c-soft` 边框。`[全选]` 视觉略与具名 region 区分（如 ghost
button + Icon）。移动端 chip 可横滑（`overflow-x: auto`）。

**a11y**：chip 行 `role="group" aria-label="按区域过滤"`，每 chip
`<button aria-pressed={selected}>`，`[全选]` `aria-label="全选/全不选"`。
区域变化朗读 `aria-live="polite"` 通报 `已显示 N 个尺寸`（节流 300ms）。

## 3. 字符串规模（实测，**S4 修正**）

- 用户面 UI 字符串（已增补 Uploader/CropCanvas 漏数）：~**130** 条
- 数据标签（PHOTO_SPECS name + 4 条 note + 5 分类）：~**33** 条
- 错误/状态消息：~**22** 条
- **区域过滤 chip 标签**（§2.11）：~8 条（6 区域 + 全选 + 搜索覆盖提示）
- **键总数估计**：v1 ≈ **230–270** 条 × 5 语种 = **~1300–1400 译条**

## 5. 风险与缓解（**新增 4 项**）

| 风险 | 缓解 |
|---|---|
| Plain-Vite (非 SvelteKit) Paraglide 集成未在我们栈验证 | P0 Gate + kill-switch 切 @nanostores/i18n |
| ja/ko 译文质量不足 | 文件标记 `pending native review`；UI 不阻塞；追迹补译 |
| 英文 UI 比中文长 ~40% | P3 每板视觉抽查 5 locale；窄列控件 `min-width: max-content`；用 pseudo-locale 拉伸测试 |
| 跨系统文件名兼容（Windows GBK / CJK 编码） | **改提供 user pref `useAsciiFilenames` 默认 off；明示触发条件，不静默回退** |
| `svelte-awesome-color-picker` 弹窗自带 EN 标签 | **明示「保持英文不译」**（plan 内决定），或用 `components` slot 重写 — v1 暂保留 |
| Cropper.js v2 chrome 默认无可见字符串 | 已验证：本项目自绘工具栏 OK |
| 生成 `src/lib/paraglide/` 入库引发 merge 冲突 | gitignored + `prebuild`/`predev` 跑 paraglide-js compile |
| Phase 3 SizeLibrary 落地时 nameKey 未补全 | 过渡 fallback `spec.nameKey ?? spec.name`，`i18n:check` 宽松模式 |
| 健康检查脚本误判 | `i18n:check` 在 Phase 4 严格化前为告警；Phase 4 起为错误 |

## 6. 完成定义（v1，**N2 N3 修订**）

- 5 locale 完整切换，chrome+panel+错误+尺寸名+文件名全译
- `svelte-check`/`oxlint`/`biome` 全净
- `i18n:check` 脚本（严格模式）通过：5 locale 键集等价
- 初始包增量 **<3KB gzip**（修订：220–260 键含运行时，1–3KB 是实际区间）
- ja/ko/zh-Hant 文件均标记 `pending native review`
- **Pseudo-locale (xx-AC) 渲染抽查**：每页 5 个最紧窄字段无截断/换行
- **真实设备 5 locale 手动冒烟**（非仅 build artifact）
- 至少在 1 个 screen reader 内确认 `aria-live` 切换声播报
- README 注明：切换方法、当前译文质量级别、跨 tab 不同步行为为预期、
  生产部署的 `<html lang>` SEO 含义
