# v0.5 研究笔记 — Alpha101 子集吸收

## 核心思想

Kakushadze (2015) 的 101 个公式 alpha 是 WorldQuant 公开的因子集，覆盖：
- 量价背离 (#3, #6, #14, #15)
- 短期反转 / 反应过度 (#4, #9, #10)
- 时序排名变换 (#1, #4, #5)
- 波动率条件信号 (#1, #7)
- 成交量方向 (#12)

挑 4 个互补的，加进我们的**信号池 (signal pool)**，让 ic_weight combiner
自动学每个 alpha + 每个 TF 的权重。

## 选定 alpha + 落地理解

### Alpha#1
```
rank(ts_argmax(SignedPower(returns<0 ? std20 : close, 2), 5)) - 0.5
```
- 用近 5 日**最大平方信号位置**作为预测
- 跌时用波动率，涨时用收盘价——本质是**风险调整后的极值时间排序**
- 反映极端波动的**时间集聚**

### Alpha#4
```
-1 * ts_rank(rank(low), 9)
```
- low 的截面 rank 在过去 9 日的时序 rank
- 取负 → **越接近 9 日 low rank 最低（即近期持续在低位）的反而预期上涨**
- 类似"超卖反弹"信号

### Alpha#6
```
-1 * correlation(open, volume, 10)
```
- 过去 10 日 open 与 volume 的相关系数取负
- **量价背离强 → 信号强**：高开伴低量（弱买盘）= 看空，低开伴高量（恐慌底部）= 看多
- 信息论角度：相关系数低意味着信息冲突，反转风险高

### Alpha#12
```
sign(delta(volume, 1)) * (-1 * delta(close, 1))
```
- 量增（前一日）× 跌价（前一日）的反向
- **量增价跌 → 看多反转；量增价涨 → 看空反转**
- 经典"放量见顶/见底"思想

## 设计选择

### 1. 算子库 `src/alphas.py`

抄 alphas101 的 `ts_*` / `rank` / `correlation` / `delta` 等基础算子（向量化版本）。

### 2. Alpha 协议

每个 alpha 是 `Callable[(WideOHLCV, ...), WideSignal]`，输入是命名 OHLCV
DataFrame 字典，输出 wide 信号。

### 3. 信号池 + ic_weight combiner

```python
compute_pool_factor(
    panel,
    alphas=['reversal_v02', 'alpha001', 'alpha004', 'alpha006', 'alpha012'],
    timeframes=[M5, M15, H1, D1],
    combiner='ic_weight',
    ic_lookback=60,
)
```

将 (alpha × tf) 笛卡尔积所有 N 个信号 → 用同一个 ic_weight signed combiner 融合。

### 4. 评估

每个信号独立跑 IC + decay，然后融合后跑 IC + decay。
看融合是否 > max(单信号)，且 turnover 是否 < max(单信号)。

## 注意事项

- alpha #1 / #6 / #12 都需要 OHLCV 全字段，不能只用 close
- 我们的 `compute_factor` 当前只接 close —— v0.5 必须升级到 OHLCV
- alpha101 原文用 daily 数据，分钟数据上时序窗口要按比例缩短或保持 bar 个数

## 不做

- 不实现 101 全部 alpha（边际收益递减，且很多互相高相关）
- 不引入 IndClass / SubIndustry（无行业数据）
- 不实现 vwap-based alpha（合成数据 vwap = close）
