# v0.6 研究笔记 — Qlib KBAR + 量价相关性家族

## 来源

[reference/qlib/qlib/contrib/data/loader.py:104-126](../reference/qlib/qlib/contrib/data/loader.py)

Qlib Alpha158 把 K 线形态拆成 9 个标量特征（KBAR family），覆盖蜡烛图所有几何要素。

## KBAR 公式表

| 名字 | 公式 | 含义 |
|---|---|---|
| KMID | (close-open)/open | 实体方向（涨跌幅） |
| KLEN | (high-low)/open | K 线全长（相对波幅） |
| KMID2 | (close-open)/(high-low+ε) | 实体占比（"决心"） |
| KUP | (high-max(open,close))/open | 上影线长 |
| KUP2 | (high-max(open,close))/(high-low+ε) | 上影线占比（高位抛压） |
| KLOW | (min(open,close)-low)/open | 下影线长 |
| KLOW2 | (min(open,close)-low)/(high-low+ε) | 下影线占比（低位接盘） |
| KSFT | (2*close-high-low)/open | close 在区间的位移 |
| KSFT2 | (2*close-high-low)/(high-low+ε) | close 区间位置归一 |

「2」结尾的版本以 (high-low) 归一化，把幅度噪声剥离，**只剩形态**——
对跨标的 / 跨 TF 比较更稳健。

## 量价相关家族（v0.6 同步引入两个）

| 名字 | 公式 | 思想 |
|---|---|---|
| CORR20 | corr(close, log(volume), 20) | 价量同步性 |
| CORD20 | corr(close/close.shift(1), log(volume/volume.shift(1)+1), 20) | 价量变化同步性 |
| RSV20 | (close-min(low,20))/(max(high,20)-min(low,20)+ε) | 区间位置（KDJ 雏形） |

## 设计选择

### 与 Alpha101 的差异
- Alpha101 公式已**带方向符号**（`-1 *` 等），单独可用
- KBAR 是**状态特征**，单独无方向——必须经过 ic_weight 让组合器学符号
- 所以 KBAR 比 Alpha101 更适合我们的 signed combiner 架构

### 形状 vs 幅度
- 「2」版本因子（KMID2/KUP2/KLOW2/KSFT2）已经被分母 (high-low) 归一化，
  跨标的尺度统一。直接进 ic_weight，不必额外 z-score。
- 非「2」版本除以 open，依然有标的间幅度差异，**combiner 内部需 cs_zscore** 处理。

### 与 v0.5 比较
- v0.5 加 4 个 Alpha101 → 池子从 1 个信号 → 5 个，IC 没显著提升
- v0.6 加 11 个 KBAR + 量价 → 池子可达 16+ 信号
- 关键测试：min_abs_ic 阈值 + 池子大小的 sweet spot
