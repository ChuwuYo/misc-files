# v0.3 研究笔记 — 多时间框架（Multi-Timeframe, MTF）因子设计

## 动机

用户 2026-04-21 强制：因子必须支持所有短线/长线时间框架。
单一日频因子无法捕捉日内反转、周线动量等多尺度信号。
MTF 设计是从专业 quant 系统（Qlib, vnpy, jeknow/Tushare 等）借鉴的标配。

## 时间框架定义

```
短线: 1m, 5m, 15m, 30m, 1h, 4h
长线: 1d, 1w, 1M
```

pandas resample 规则映射：

| TF | pandas rule | 每日 K 数（日历日） |
|---|---|---:|
| 1m | "1min" | 240 (A股) / 390 (US) / 1440 (crypto) |
| 5m | "5min" | 48 / 78 / 288 |
| 15m | "15min" | 16 / 26 / 96 |
| 30m | "30min" | 8 / 13 / 48 |
| 1h | "1H" | 4 / 6.5 / 24 |
| 4h | "4H" | 1 / ~1.6 / 6 |
| 1d | "1D" | 1 |
| 1w | "1W-FRI" | — |
| 1M | "1ME" | — |

## 核心文献 / 工程参考

### 多时间框架动量
- **Moskowitz, Ooi, Pedersen (2012)** "Time Series Momentum" (JFE) — 跨 1-12 月 horizon 的时序动量。启发：HTF 用动量，LTF 用反转，符号正好相反。
- **Han, Zhou, Zhu (2016)** "A trend factor" — 多窗口 MA 信号融合。

### 多时间框架反转
- **Da, Liu, Schaumburg (2014)** "A Closer Look at the Short-Term Return Reversal" — 拆分隔夜 vs 日内反转，证明日内反转更稳。

### 工程实现
- **Qlib** `qlib/contrib/data/handler.py` — `Alpha158` 默认日频但 handler 接口支持任意 freq；`qlib.data.D.features(freq='1min')` 直接给分钟级特征。
- **vnpy** `vnpy_ctastrategy/template.py` — `BarGenerator` 多窗口聚合器，K 线增量合成
- **TA-Lib / pandas-ta** — 所有指标天然 freq-agnostic

## 设计选择

### 1. 抽象层

```
TimeFrame (Enum)         # 1m, 5m, 15m, 30m, 1h, 4h, 1d, 1w, 1M
BarStore                 # 持有 base TF（最细粒度）的 OHLCV，按需聚合
resample_to(tf)          # OHLCV-aware 聚合：open=first, high=max, low=min, close=last, vol=sum
```

### 2. 因子接口扩展

```python
compute_factor(panel, timeframe='1d', ...)        # 单 TF
compute_mtf_factor(panel, timeframes=[...], combiner='ic_weight')   # 多 TF 融合
```

### 3. 跨 TF 信号组合策略

| 组合方式 | 描述 | 何时用 |
|---|---|---|
| `equal` | 各 TF 信号 z-score 后等权平均 | 简单基线 |
| `ic_weight` | 各 TF 历史 IC 滚动加权 | 自适应，推荐 |
| `htf_filter` | HTF 信号决定方向，LTF 调整强度 | 趋势 + 时机 |
| `regime_switch` | 按波动率制度切换主导 TF | 高级，留 v0.x |

v0.3 实现 `equal` 与 `ic_weight` 两种。

### 4. 评估扩展

- 每 TF 独立 IC 报告
- 跨 TF 信号相关性矩阵（避免冗余融合）
- 各 TF 适配的 forward horizon（1m TF → 1m horizon，1d TF → 1d horizon）

## 不做（暂缓）

- Tick 级（< 1 分钟）
- 自适应 TF 选择（让模型挑 TF）—— 留 v1.x ML 阶段
- 跨 TF 因子残差化（PCA on MTF signals）
