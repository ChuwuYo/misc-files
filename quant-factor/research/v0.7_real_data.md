# v0.7 研究笔记 — 真数据接入

## 数据源选择

| 源 | 市场 | 频率 | 认证 | 速度 | 备注 |
|---|---|---|---|---|---|
| **yfinance** | 全球（主要 US） | 1m/5m/15m/30m/1h/1d | 无 | 中 | 1m 限近 7 天，5m 近 60 天 |
| akshare | A 股 | 日线为主 | 无 | 慢 | 国内研究首选 |
| Tushare | A 股 | 全频 | 需 token | 快 | 商业付费 |
| ccxt | 加密 | tick / 1m / ... | 视交易所 | 快 | crypto 友好 |

**v0.7 选 yfinance**：
- 无 token 即可用，本地立即可跑
- 美股大盘，流动性好，因子信号"现实"
- daily 历史 5 年+，足够 IC 显著性

A 股 / 加密 留 v0.x 通过同套 panel 接口扩展（adapter 模式）。

## Universe 设计

S&P 100 头部 50 只（科技 + 金融 + 消费 + 能源 + 医疗）。
理由：
- 流动性高 → 数据干净
- 板块分散 → 因子不偏向单一行业
- 数量适中（50）→ 截面 IC 估计稳定 + 计算可控

排除：
- ADR / 双重上市（数据复杂）
- 上市 < 3 年的（缺历史）
- 高波动垃圾股（噪声大）

## API 调用模式

```python
yf.download(
    tickers=['AAPL', 'MSFT', ...],   # 多 ticker batch
    period='3y',                      # 或 start/end
    interval='1d',                    # 1m/5m/.../1d
    progress=False,
    auto_adjust=False,                # 我们要 raw OHLCV
    threads=True,                     # 并发
)
```

返回 MultiIndex(field, ticker) wide 格式——需 melt 到我们的 long panel 接口。

## 与合成数据的差异（预期）

| 维度 | 合成 | 真实预期 |
|---|---|---|
| KBAR pool IC | 0.386 | **0.02 ~ 0.05** |
| Half-life | 3.2 hr | **3 ~ 10 天** |
| OOS = IS | 是 | **OOS 通常 < IS**（市场制度变化） |
| t-stat | 100+ | 2 ~ 5 |
| Turnover | 12% | 30 ~ 60% |

**心理准备**：合成数据 IC=0.386 是"完美数据 + 完美算法"——真数据上 IC=0.05 已是顶尖因子。

## 风险

1. **网络稳定**：yfinance 偶有 429 / 超时，需重试
2. **字段对齐**：yfinance 用 `Adj Close` vs `Close`——我们要 raw OHLC，统一用 close（不复权）然后看是否有 corporate action 影响
3. **涨跌停 / 停牌**：US 股票几乎无停牌；A 股需特殊处理
4. **Look-ahead**：避免使用 fwd-adjusted 价格（dividends future-adjusted）

## 不做（暂缓）

- 真实 1m / 5m intraday 数据（yfinance 限 60 天，留 v0.x）
- 多市场拼接（先做美股，再扩 A 股 / 加密）
- 复权处理（暂用 raw close，v0.x 加复权）
