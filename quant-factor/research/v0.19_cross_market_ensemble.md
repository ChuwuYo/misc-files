# v0.19 研究笔记 — 跨市场 ensemble

## 动机

Binance OI 历史仅 30 天，walk-forward 用不了。Pivot：用现有 SP50 (美股) +
Crypto (Binance) 两个相对独立的市场，做 portfolio-level 信号 ensemble。

## 假设

**经济直觉**：
- 美股 daily 最优 horizon = 10/20 天（中期反转）
- Crypto daily 最优 horizon = 20/30 天
- 两市场不同时区、不同 macro 驱动、不同投资者群体 → 因子收益时间序列**低相关**

**预期**：portfolio-level Sharpe ≈ √2 × max(单市场 Sharpe)
- SP50 walk-forward agg_t = +3.27 (h=20)
- Crypto walk-forward agg_t = +3.59 (h=20, with funding)
- **ensemble 预期** agg_t ≈ +4.5 to +5.0（依据 Sharpe 加性公式）

## 实施路径

### 1. 各市场独立计算因子
不混合 universe（不同 universe 截面无可比性）。
SP50 跑 v0.11 winner，Crypto 跑 v0.18 winner。

### 2. portfolio Sharpe 算法

```python
# 每市场各自构造等权 long-short portfolio
# top quintile - bottom quintile, 等权 within quintile
sp_pnl_daily = factor_to_pnl(sp_factor, sp_close)   # daily series
ct_pnl_daily = factor_to_pnl(ct_factor, ct_close)   # daily series

# 均匀对齐（缺数据日填 0 = no trade）
joint_idx = sp_pnl.index.union(ct_pnl.index)
sp = sp_pnl.reindex(joint_idx).fillna(0)
ct = ct_pnl.reindex(joint_idx).fillna(0)

# 50/50 capital allocation
combined = 0.5 * sp + 0.5 * ct
sharpe(combined) vs sharpe(sp), sharpe(ct)
```

### 3. 三个核心指标
- 单市场 Sharpe（年化）
- ensemble Sharpe（年化）
- 跨市场 PnL 相关系数（应 < 0.2 才能验证假设）

## 不做（暂缓）

- IC-based portfolio weights（非常数权重）—— 简单 50/50 baseline 优先
- 杠杆 / vol target — 留 portfolio layer
- 同一 universe 的多 factor stack — 另一个方向
