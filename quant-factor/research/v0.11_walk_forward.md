# v0.11 研究笔记 — Walk-Forward 滚动验证

## 动机

v0.7-v0.10 用 50/50 IS/OOS 一刀切。问题：
- 只有**一个**OOS 估计，无置信区间
- OOS 期与 IS 期处于不同**市场制度**——一次评估容易把制度差异误读为 alpha
- 真实生产的因子是**rolling fit**：每天用过去 N 天数据更新模型，预测未来

Walk-forward 给出多个 OOS 子段，可统计 IC 跨段稳定性。

## 核心文献

### Bailey & López de Prado (2014) "The Deflated Sharpe Ratio"
- 多重检验下的 Sharpe 校正——指出 backtest 过拟合的统计学根源
- 推论：单一 OOS 估计的置信区间需用 ≥ K 个独立 OOS 子段才能给出

### López de Prado (2018) "Advances in Financial Machine Learning" Ch.7
- **Combinatorial Purged Cross-Validation (CPCV)**：
  - 把时间序列切成 N 个等长块，从中取 K 块作 test，剩余 N-K 块作 train
  - C(N,K) 种组合，每种给出一个 OOS IC
  - 对**purge**：剔除 test 与 train 之间的"信息泄漏区间"（forward return horizon）
  - 对**embargo**：test 后再隔几天才让 train 重启
- 解决：单 OOS 的方差估计 + 防止 horizon 重叠泄漏

### 工程经验
- 学术研究常用 5/10 折 CPCV
- 产业更常用 **anchored walk-forward**：train 窗口逐步扩张，test 窗口固定大小滑动
- 我们 SP50 × 3y = 750 天，可分 4-6 段，每段 ~125-187 天

## 设计选择

v0.11 实现 **anchored walk-forward**（最简）：

```python
walk_forward_evaluate(
    panel,
    factor_fn,           # callable: (train_panel) -> wide factor
    n_splits=5,
    min_train=180,       # 至少 180 天 warmup
    test_size=120,       # 每个 test 窗 120 天
    purge=10,            # train 末尾留 10 天 purge gap
    horizon=10,
)
    -> per-fold IC table + aggregate stats (mean, std, t)
```

实现概要：
1. 按时间排序所有日期
2. 第 i 折：
   - train_end = min_train + i * test_size
   - train = panel[date < train_end - purge]
   - test = panel[train_end <= date < train_end + test_size]
   - factor = factor_fn(train + test)  # factor 自己内部确保不窥探未来
   - 评估 factor[test 时段] 的 IC
3. 收集所有折的 IC，输出均值 ± 标准差 + t 统计

## 实施细节

### Purge 与 horizon
forward return horizon=10 意味着 day t 的 fwd_ret 用到 day t+10 价格。
若 train 末日是 T，test 起始也用到 T+1...T+10——重叠区间需 purge：
**train cutoff = test_start - horizon**。

### Combiner 的 lookahead
v0.9 的 ic_weight 用 rolling IC + shift(1)——已无 lookahead。
PCA 用全样本 SVD——**有 lookahead**，walk-forward 内部不能用全样本 SVD。
**修复**：PCA 必须只用 train 数据 fit，test 数据用 train 的 PCA basis transform。
**但当前 pca_denoise_signals 实现是 in-place**——v0.11 暂用整个 walk-forward window
作为"伪样本外"，明确标注此限制，留 v0.12 拆分 fit/transform。

## 不做（暂缓）

- CPCV（C(N,K) 折太多，计算成本高，留 v0.x）
- Rolling-fit PCA（要拆 fit/transform，留 v0.12）
- 区间 bootstrap CI（统计上更严，但 walk-forward 本身已给方差估计）
