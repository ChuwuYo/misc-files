## 关键洞见

### 1. 真"非 OHLCV"alpha
v0.5（Alpha101）、v0.6（KBAR）都是 OHLCV 衍生 —— 它们彼此高度相关，
ortho/PCA 后边际 IC 提升有限。
funding 是**全新的信息维度**（多空持仓比的派生量），与 OHLCV 几乎正交。
combiner 不需要复杂处理就能榨出价值。

### 2. ic_weight signed combiner 自动学方向
funding 在不同时段方向可能反转（牛市 vs 熊市拥挤的反应不同）。
ic_weight 的滚动 IC 自适应学符号 —— 测试中 funding 大多数时间得正权重，
少数 fold 短暂转负。

### 3. Bug 教训：alpha 注册表与默认组的耦合
`ALPHA_GROUPS['all'] = list(ALPHA_REGISTRY.keys())` 在 module load 时评估。
新增 alpha 后，'all' 自动包含新 alpha → 测试 "all + new" 实际是 "all + 重复"。
**v0.x 应该**：把 'all' 改成动态返回 / 或显式列出 alpha 名字避免歧义。

### 4. 验证流程教训
本轮第一次跑：17 vs 19 都得 IC=+0.055（identical）→ 意识到对照组有 bug
第二次跑：正确分组后真实 IC 跃升 → 真假信号一目了然
**调试时永远要对比 byte-level diff**，不要只看 IC 数字。
