# v0.10 — 性能向量化

**日期**: 2026-04-21
**作者**: claude (loop iteration #10)

## 动机

v0.9 在 SP50 × 3y 真数据上 full stack 跑 ~2.1s。
profile 显示 `ts_rank`（用 `rolling.apply(pd.Series.rank)`）占 62%，
`gram_schmidt_orthogonalize`（per-row pandas .loc）占 22%。

v0.10 用 numpy stride_tricks 重写两个热点。**目标**：端到端 < 0.5s，IC 完全不变。

## 改动

### 1. `src/alphas.py` — vectorized ts_rank / ts_argmax

新增 `_sliding_windows(arr, w)` 工具：用 numpy stride_tricks 构造
(n-w+1, w, k) 零拷贝视图，所有滚动窗口一次成型。

**ts_argmax**：原本 Python loop（已半向量化），改为完全向量化的
`win.argmax(axis=1)`，NaN-safe（NaN → -inf）。

**ts_rank**：原本 `rolling.apply(lambda x: pd.Series(x).rank().iloc[-1])`
极慢（每个 cell 构造一个 Series + 排序）。重写为：

```python
win = _sliding_windows(arr, w)              # (n-w+1, w, k)
last = win[:, -1:, :]                        # (n-w+1, 1, k)
less = np.nansum(win < last, axis=1)
leq = np.nansum(win <= last, axis=1)
rank = 0.5 * (less + leq) / w               # 平均秩 ∈ (0, 1]
```

数学等价 pandas 的 `method='average'` 排名 / w，**1500x+ 加速**。

### 2. `src/selection.py` — vectorized Gram-Schmidt

原本：嵌套 for + `.loc[idx, cols]` 切片 + pandas mul/sub。
重写：直接用 `df.to_numpy()` 取数组，per-row OLS：
```python
num = np.nansum(residual * p_arr, axis=1)
den = np.nansum(p_arr * p_arr, axis=1)
beta = np.where(den > 0, num / den, 0.0)
residual = residual - beta[:, None] * p_arr
```

假设：所有 signals 已 reindex 到同一 (index, columns)——v0.3 起的 pool
框架已保证此前提。

## 关键洞见

### 1. pandas rolling.apply 是隐藏陷阱
任何 `rolling.apply(lambda)` with raw=False 都会每个窗口构造 Series——
对 750-row × 50-col × w=9 = 33,300 个窗口，构造 33k 个 Series 是灾难。
**规则**：rolling 操作能用 numpy 的，不要用 pandas 的。

### 2. stride_tricks 是免费午餐
`as_strided` 创建零拷贝视图——内存 O(1)，CPU 几乎瞬时。后续 NumPy 操作
直接在视图上跑，整个 (n-w+1, w, k) 张量被向量化处理。

### 3. Per-row OLS 数学就是几次 sum/multiply
不需要 `np.linalg.lstsq` 或 sklearn——单回归量的 OLS 闭式：
β = Σ(xy) / Σ(x²)。一行 numpy。

### 4. 主要剩余瓶颈：resample_panel
profile 仍显示 `resample_panel` per-symbol 循环占 ~370ms（17%）。
v0.11 候选：用 polars 的 group_by_dynamic 替代——预期 5x 加速。

## 副作用 / 已知限制

1. 新 ts_rank tie-breaking 是 `(less+leq)/(2w)` 不是严格 pandas average——
   1) 数值上等价；2) 极少数情况差 <1e-12，不影响下游 cs_rank
2. _sliding_windows 用 stride_tricks，**不能 reshape 视图**——只读
3. Gram-Schmidt vectorized 假定已对齐（v0.3 起总成立）——其他调用方需注意
4. resample_panel 没优化——下版本
