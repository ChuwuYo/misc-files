# v0.16 — Intraday H1 Crypto MTF + **两次发现 & 修复 Lookahead Bug**

**日期**: 2026-04-21
**作者**: claude (loop iteration #16)

## 本轮净收获：发现并修了 MTF 的重大 lookahead bug

v0.15 假设 "intraday 微观结构可能绕过 sector exposure"。但测试的初步结果荒诞
（H1+H4+D1 MTF IC=+0.33, t=+60）让我审计了 MTF 代码路径，**发现两阶段 lookahead**：

### Bug #1：intraday TFs（H4/H1 等）默认 left-label
pandas `resample('4H')` 默认 `closed='left', label='left'`：H4 bar 标签在窗口
**起点**（如 00:00），但其 OHLCV 聚合来自 **00:00-03:59** 的 4 根 H1 bar。
当 ffill 到更细的 H1 网格，H1=01:00 拉到 H4=00:00 → 使用了 02:00-03:59 的未来数据。

### Bug #2：**D1 也是 left-labeled**
修 bug #1 后初次重跑：H1+H4 IC 回到合理的 +0.01（D1 未参与），但 **H1+H4+D1** 仍
爆出 +0.35 —— 提示 D1 作为 coarser TF 时同样泄漏。诊断：pandas 'D' 默认 left-label，
一个 "Monday" 的 D1 bar 包含 Monday 00:00-23:59 全天数据。ffill 到 H1=Monday 01:00
→ 使用了 02:00-23:59 的 23 小时未来。

### 最终修法：MTF 对齐阶段统一 shift(1)

```python
LEFT_LABELED_TFS = {M1, M5, M15, M30, H1, H4, D1}   # pandas 默认 left

def _align_to_finest(signals, finest_index, finest):
    for tf, sig in signals.items():
        if tf is not finest and tf in LEFT_LABELED_TFS:
            sig = sig.shift(1)        # ← 等 bar 关闭后才放行
        out[tf] = sig.reindex(finest_index).ffill()
```

**为什么不直接改 resample label？** 曾尝试过：会让 D1 作为 **finest** 时（SP50 D1+W1）
所有索引错位一天，进而把 SP50 IC 从 +0.022 翻成 -0.024。
**正确的层次**：保持 pandas resample 默认行为 + 在 MTF 对齐时 shift。

W1 / MO1 不在 LEFT_LABELED_TFS 里（pandas 默认 right-label）→ 不 shift ✓。

## 核心洞见

### 1. Intraday H1 alpha 是真的，但 magnitude 比 Daily 小
原假设"intraday 比 daily 更强"——**证伪**。
- Crypto D1 h=20 IC=+0.046, t=+8.16
- Crypto H1 h=24 IC=+0.010, t=+4.16

daily 强 4x。原因：H1 数据 1y 太短（8760 bar），噪声多；daily 3y 给更多 breadth。

### 2. D1 在 H1 MTF 里是"宏观上下文"
H1+H4+D1 比 H1 only：IC 从 +0.014 → +0.018（**+30% 提升**）。
解释：D1 提供慢变的"这周趋势"，与 H1 的短期反转互补。
H4 反而**没带价值**（H1+H4 IC 低于 H1 only）—— H1 已捕捉 4h 信息。

### 3. Walk-forward 本应早暴露这个 bug
v0.3 v0.6 那些"IC=0.386"的合成 intraday MTF 结果——**全部有 lookahead**。
walk-forward 本身并不能自动暴露 lookahead，因为泄漏是 intra-bar 性质的、
每个 test fold 都同样污染。必须通过**代码审计**发现。

## 副作用 / 已知限制

1. shift(1) 让 intraday HTF 信号 **晚 1 个 HTF bar 才可用** —— 不是严格意义上"当前
   bar 关闭的瞬间"可用，而是"下一个 bar 开始时才可用"。生产级可能希望用
   HTF_bar_close_timestamp + small epsilon 的精细对齐，但当前足够
2. 修 bug 后 intraday IC 本来就比 daily 弱，H1-H4 组合没有显著价值——说明
   本项目的 alpha 栈在**日级/中频**更强，不适合做短线 intraday quant
3. 此 bug 在所有合成 intraday 测试（v0.3-v0.6）中都存在，那些数字不能对外引用
