## 动机

立项第一版。先用最经典、最被验证过的信号——Jegadeesh (1990) 短期反转——
铺设端到端流水线（数据 → 因子 → 回测 → 报告）。后续每轮在这个基础上换零件。

## 改动

新建仓库骨架：

- `src/factor.py` 实现 `compute_factor(panel, window=5)`：log return 累加 → 截面 rank → 取负号
- `src/data.py` 合成数据生成器，内嵌可控反转信号（`reversal_strength=0.05`）以便离线验证
- `backtest/evaluate.py` 标准评估套件：IC / Rank IC / IR / Top-Bottom 年化 / 换手率
- `README.md` + `ROADMAP.md` + `research/sources.md`

## 解读

- 合成数据的反转信号被注入在 t-1 → t，因此 window=1 是「作弊式贴合」，
  IR=5 不可对外宣传。把 window=5 作为汇报口径更诚实。
- Top-Bot 年化在 window=10 反弹，提示**多窗口加权**可能优于单窗口（v0.4 验证）。
- 换手率随窗口拉长单调下降，符合预期。
- 流水线串通，各阶段可拔插。

## 副作用 / 已知限制

1. 合成数据的反转过强且无风险因子干扰，**真实数据上 IC 必然衰减**——只是为了打通流程。
2. 没有任何中性化（行业/市值/Beta），真实数据下信号会被风格污染。
3. NaN 处理简单：rank 时整行 NaN 直接跳过，未做缺失填补策略。
4. `pd.qcut` 在每行重复值多时会丢分组，目前用 `duplicates='drop'` 忍受。
