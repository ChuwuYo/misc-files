# v0.5 — Alpha101 子集 + 信号池 + Qian-Hua 稀疏化

**日期**: 2026-04-21
**作者**: claude (loop iteration #5)

## 动机

到 v0.4 我们的 ic_weight combiner 只融合「同一信号 × 多 TF」。
v0.5 把架构升级到「**多信号 × 多 TF**」的笛卡尔积信号池——任何 alpha 都可以加入池子，
combiner 自动学其权重和符号。

吸收 4 个互补的 Alpha101 子集（#1 / #4 / #6 / #12）作为试金石。

研究：[research/v0.5_alpha101_subset.md](../research/v0.5_alpha101_subset.md)
开源参考：[reference/WorldQuant_alpha101_code/](../reference/WorldQuant_alpha101_code/)

## 改动

### 1. 新建 `src/alphas.py`

向量化算子库（直接对应 Alpha101 论文）：
- `ts_sum / ts_mean / ts_std / ts_min / ts_max / ts_argmax / ts_rank`
- `delta / delay / correlation / signed_power / scale / cs_rank`

实现 Alpha101 子集：
| 函数 | 公式 | 思想 |
|---|---|---|
| `alpha001` | `rank(ts_argmax(SignedPower(returns<0?std20:close, 2), 5)) - 0.5` | 波动率条件极值时序排名 |
| `alpha004` | `-1 * ts_rank(rank(low), 9)` | 持续低位反弹 |
| `alpha006` | `-1 * correlation(open, volume, 10)` | 量价背离反转 |
| `alpha012` | `sign(delta(volume, 1)) * (-1 * delta(close, 1))` | 量增价跌看多 |
| `reversal_v02` | v0.2 vol-dampened reversal 包装为 alpha | 我们自己的基线 |

注册到 `ALPHA_REGISTRY` 字典，按名字调度。

### 2. `compute_pool_factor` 函数

```python
compute_pool_factor(
    panel,
    alphas=['reversal_v02', 'alpha001', 'alpha004', 'alpha006', 'alpha012'],
    timeframes=[M5, M15, H1],
    combiner='ic_weight',
    ic_lookback=60,
    min_abs_ic=0.05,            # Qian-Hua 稀疏化：低于阈值的 slot 权重清零
)
```

笛卡尔积 N=`len(alphas) * len(timeframes)` 个信号 → 同一个 ic_weight signed combiner。
**新增 `min_abs_ic` 阈值**：滚动 IC 绝对值低于阈值的 slot 直接 weight=0，杀噪声。

| alpha001 | -0.0020 | -0.29 | -1.4 |

**所有 Alpha101 在合成数据上接近噪声**（|t| < 1.5 全部）。

alpha101 设计针对的市场微结构信号无从检测——意料之中。

## 关键洞见

### 1. 框架 ≠ 收益
v0.5 的信号池架构是正确的，但**合成数据上 alpha101 不贡献**。
真数据（v0.7 后）才能真检验它们的边际价值。

### 2. 稀疏化是必须的
在大池子中，弱 / 噪声 slot 会通过 ic_weight 的 L1 归一化偷走权重。
`min_abs_ic` 阈值是 Qian-Hua 风格的最简稀疏化——更进一步可以加 LASSO / IC clustering。

### 3. 性能瓶颈
17s 跑 460k 行。`ts_argmax` 用 Python loop（O(n×w×k)），是大头。
**v0.10 优化目标**：用 numba 或 polars rewrite ts_argmax / ts_rank。

### 4. 包装一致性问题
`reversal_v02` alpha 返回 raw 值，没经过 winsorize / cs_rank / neg pipeline，
所以池中 reversal 信号弱于 MTF 中的 reversal——下版本统一 alpha 输出
约定（例：必出 z-score 或 cs_rank）。

## 副作用 / 已知限制

1. ts_argmax 慢——见性能瓶颈
2. alpha006 / alpha012 需要 volume，合成 volume 是 lognormal 噪声，无信号
3. 未实现 vwap / IndClass / market 等 Alpha101 高级算子（依赖真数据）
4. min_abs_ic 是全局静态阈值——理想是按 slot 的历史 IR 分位数自适应

## 与开源因子的对照

| 维度 | v0.5 本地 | Alpha101 原版 | Qlib FactorComposer |
|---|---|---|---|
| 多 alpha 支持 | ✓ 注册表 + 池子 | ✓ 类方法 | ✓ Handler 配置 |
| 多 TF 笛卡尔积 | ✓ | ✗ | ✗（需手动） |
| 自适应符号学习 | ✓ signed ic_weight | ✗（公式固定符号） | ✗ |
| 稀疏化 | ✓ min_abs_ic | ✗ | ✗（独立筛选） |
| 算子向量化 | ✓（ts_argmax 除外） | ✓ | ✓ |

我们的差异化在「**自适应权重 + 稀疏化**」，比 Alpha101 公式手工调更省人力。
