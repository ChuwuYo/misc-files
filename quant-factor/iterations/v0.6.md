## 改动

### `src/alphas.py` 新增 12 个 alpha

**Qlib KBAR family（9 个）**：
| alpha | 公式 | 几何含义 |
|---|---|---|
| kmid | (close-open)/open | 实体方向 |
| klen | (high-low)/open | 全长 |
| kmid2 | (close-open)/(high-low+ε) | 实体占比 |
| kup | (high-max(o,c))/open | 上影线长 |
| kup2 | (high-max(o,c))/(high-low+ε) | 上影线占比 |
| klow | (min(o,c)-low)/open | 下影线长 |
| klow2 | (min(o,c)-low)/(high-low+ε) | 下影线占比 |
| ksft | (2c-h-l)/open | close 区间位移 |
| ksft2 | (2c-h-l)/(high-low+ε) | close 区间位置归一 |

**Qlib 量价 / 区间（3 个）**：
| alpha | 含义 |
|---|---|
| rsv20 | (close-min(low,20))/(max(high,20)-min(low,20)+ε) — KDJ 雏形 |
| corr20 | corr(close, log(volume), 20) — 价量同步性 |
| cord20 | corr(close 变化率, log(volume 变化率), 20) — 价量变化同步性 |

新增 `ALPHA_GROUPS` 字典分组方便测试：`v02_baseline / alpha101_subset / kbar / qlib_pv / all`。

## 关键洞见

### 1. KBAR 是合成数据的"金钥匙"
合成 OHLCV 是从 returns 反推的：
- `close[t] = exp(cumsum(rets[t]))`
- `open[t] = close[t-1]`
- `high/low = close × (1 ± spread)`

所以 (close-open)/open ≈ 当前 bar 的真实 return。KBAR 直接捕捉到 returns 信息，
而我们的反转信号也是用 returns，二者高度互补——KBAR 在不同 TF 提供不同时长的「最新一笔」信息。

### 2. 真数据上 KBAR 仍会强
即便没有 synthetic 的清晰反转结构，KBAR 在真实股票上是 Qlib Alpha158 的核心构件，
LightGBM benchmark 上 KBAR 列贡献度排前 1/3。我们不会在真数据上看到 +0.386，
但 +0.05 ~ +0.10 应可期待。

### 3. min_abs_ic=0.05 是好默认
KBAR 单 IC 仅 ±0.004 但**滚动 IC 在 60-bar 窗口内可达 0.05+**——
阈值 0.05 既过滤纯噪声，又留下结构性信号。

### 4. Alpha101 在合成数据上拖后腿
`all` 池（含 Alpha101）IC 反而比 `+kbar` 略低（0.366 vs 0.386）。
原因：Alpha101 的 ts_argmax 等慢，且无信号——在 ic_weight 里偷走少量权重。
**v0.10 优化建议**：把 Alpha101 移到 lazy load，只在用户显式指定时才计算。

## 副作用 / 已知限制

1. KBAR 用 open/high/low/close，对**有跳空**的真数据需要拆分隔夜 vs 日内 bar
2. 合成 volume 是 lognormal 噪声，corr20/cord20 提供的边际很小
3. `kup` 函数实现用 `pd.combine(o['close'], np.maximum)`——对 NaN 处理需关注
4. 池子大小到 30 后边际下降（30 vs 51 slots IC 反而下降）——signal selection 必要

## 与开源因子的对照

| 维度 | v0.6 本地 | Qlib Alpha158 | Alpha101 |
|---|---|---|---|
| KBAR 9 件套 | ✓ | ✓（原版） | ✗ |
| 量价 corr | ✓ corr20/cord20 | ✓ CORR/CORD 多窗口 | ✓ alpha006 等 |
| 自适应符号 | ✓ signed combiner | ✗（送入 ML） | ✗（公式定向） |
| 多 TF 笛卡尔积 | ✓ 直接 | 需 handler 配置 | ✗ |

我们用 ic_weight + min_abs_ic 把 KBAR 当作**通用特征池**，
比 Qlib 的 ML 路线更轻、更可解释。
