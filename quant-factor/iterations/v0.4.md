## 改动

### evaluate.py 扩展

1. **`FactorReport` 增字段**：`rank_ic_mean`, `rank_ic_ir`, `t_stat`
2. **`_row_rank_corr`**：Spearman = Pearson on per-row ranks（截面异常值鲁棒）
3. **`ic_decay(factor, close, horizons)`**：跨 horizon IC 表，含 Pearson、Spearman、IR、t-stat
4. **`ic_half_life(decay)`**：log-linear 拟合 IC(h) = IC(1)·exp(-λh)，返回 ln(2)/λ
5. **`ic_autocorr(factor, close, max_lag)`**：每日 IC 序列的自相关（lag 1-20）
6. **`evaluate_horizons(...)`**：unified 报告 = decay + half-life + autocorr

**半衰期 ≈ 52 bars = 260 min ≈ 4.3 小时**——信号缓慢衰减，调仓窗口宽。

**IC autocorr lag 1-5**: -0.15 / -0.15 / +0.23 / -0.14 / -0.14 → 全部 < 0.25，**接近纯 alpha**，

## 关键洞见

### 1. IC 峰值不在 h=1 — 「慢热信号」
合成数据反转在 h=2-3 IC 比 h=1 还高（+0.156 → +0.189）。这是因为：
- h=1 forward return 包含日内噪声
- h=3 forward return 累积了 reversal 的完整生效路径

**实务含义**：不要盲目用 h=1 做调仓，看完整 decay 曲线选峰值或拐点。

### 2. 半衰期 = 持有期建议
半衰期 4.3 小时 → 持仓 4-8 小时最优，超过会冲淡 alpha；少于 1 小时浪费 alpha。

### 3. t-stat 决定可信度
日数据 t=1.5 → 不显著（< 1.96 阈值），需更多数据或更强信号。
分钟数据 t=80+ → 极显著但要警惕：合成数据自然过度显著，真数据更接近 t=2-5。

### 4. Rank IC vs Pearson IC
当二者差距大时说明因子分布尾部异常重，应优先看 Rank IC。
我们的 ic_weight 输出 [-1,1] 截面 rank，所以二者天然接近。

## 副作用 / 已知限制

1. `ic_half_life` 在 IC 不单调衰减（如先升后降）时只用单调下降部分拟合——
   合成数据上半衰期取自 h=3 之后下降段
2. `ic_autocorr` 用日 IC，对高频 IC 的 intra-day pattern 看不到——可加 hour-of-day 分组
3. 没有报告 IC 在不同 quantile 的差异（top quintile IC 通常更高）——v0.5 加
