### 为什么数学预测失败

数学上 √2 lift 假设两 PnL：
- 几乎独立 ✓ (verified: corr ≈ 0)
- **vol 相近** ✗（crypto vol 是 SP50 的 2.1x）
- **mean Sharpe 相近** ✗（SP +2.63 vs Crypto +0.85 walk-forward）

加 0 相关但低 Sharpe 的 Crypto 流，实际上是把 SP 这个稳信号"稀释"为含 noise 的混合
信号。Diversification 数学预测的是 SAME-Sharpe 资产的 √2 lift，不适用于此。

### 为什么 Crypto walk-forward Sharpe 差（+0.85 vs IS-Sharpe +2.48）

v0.18 报告 Crypto walk-forward IC agg_t=+3.59，但 portfolio Sharpe 才 +0.85。
**IC vs Sharpe 区别**：
- IC = 因子值与未来收益的截面相关 → 衡量**排序能力**
- Sharpe = portfolio return / vol → 衡量**实际经济价值**

Crypto 因子排序能力 OK 但 portfolio vol 极高（103% ann），加上 fold 0/1 处于
2024H1 大震荡期，Sharpe 表现远不如 IC 表现优雅。

**生产级教训**：评估因子时**两个指标都要看**。IC 高 + Sharpe 低 = 信号正确但
做不出钱（vol 太大、universe 太集中或制度切换风险高）。

## 关键洞见（即使假设失败）

### 1. 跨市场 PnL 真低相关（首次量化）
SP50 与 Crypto 的因子 PnL 时序相关 ≈ 0。这是 **portfolio 视角下** 的真实独立性
确认。如果未来有方法把 Crypto 的 Sharpe 拉起来（如更稳的 alpha），ensemble
理论上会工作。

### 2. SP50 因子在 walk-forward 上**比之前 IC-based 评估显示得更稳**
Sharpe-based walk-forward 显示 SP50 100% win-rate，mean +2.63。这与 v0.11
的 IC walk-forward agg_t +4.39 一致（高 IC + 低 universe vol = 高 Sharpe）。

### 3. Crypto-only 不适合作为独立 portfolio
walk-forward Sharpe +0.85, 60% win-rate, std 2.17。fold 0 (-2.22)、fold 1 (-1.35)
说明 crypto 因子在某些制度（2024 H1 BTC 暴涨期）会失效。

### 4. portfolio.py 是真新基础设施
现在能算实际 Sharpe / DD / win-rate / 跨市场组合 —— 比单 IC 更接近真实交易表现。
未来 v1.0 文档应推 Sharpe-based 评估为主。

## 副作用 / 已知限制

1. portfolio_to_pnl 用等权 quintile，没考虑交易成本 / 滑点
2. ensemble 用日线对齐，crypto 24/7 vs SP50 252-day 错位但 fillna(0) 处理
3. adaptive gate (+2.79 vs SP-only +2.63) 看似最佳但只 80% win-rate，**std 提升**
   不一定值得
4. walk-forward 5 折，单 fold 100 天 — fold 间制度差异主导样本，建议 10+ 折更稳
