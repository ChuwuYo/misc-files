**Crypto 最优**：lookback=180, ewma=0（无 smoothing）。

- crypto 制度切换快 → lookback 长（180）平均化制度噪声
- crypto 反转快 → smoothing 反而吃掉信号峰（与 equity 慢反转相反）

## 关键洞见

### 1. 每市场超参不同，但架构通用
SP50 与 Crypto 都需 ortho + ic_weight + D1/W1，但 lookback / ewma 几乎对称翻转。
**生产建议**：每个市场独立 walk-forward 调超参，不要假设 portable 默认值。

### 2. Crypto 信号峰更短促但更频繁
h=3 是 crypto 独有的强信号点（equity 在 h=3 仅 +0.013, t=+1.24）。
crypto 微观结构（taker / maker / liquidation cascade）让短期反转更显著。

### 3. SP50 vs Crypto 长 horizon 信号都强
h=20-30 在两个市场都正显著——支持"中期延续"是普适现象（不分资产类别）。

### 4. 24/7 vs 252-day calendar
crypto 1096 obs vs SP50 751 obs 给 crypto 更多 fold 自由度，但**单 fold IC std 更高**——
说明 crypto 真实噪声 > equity，bonferroni 校正下需要更多观察才能保 t-stat。

## 副作用 / 已知限制

1. Binance 公共 API rate-limit（6000 weight/min）—— 拉 50 个币顺序需 ~21s，并行可降到 5s
2. BINANCE_TOP50 是手工策展，未含 dynamic universe 调整（如新上市 / 退市）
3. **没有 sector 分类**（L1 / L2 / DeFi / Meme / Storage / Privacy）—— v0.15 候选
4. 仅 daily，未测 H1 / M15（Binance 都免费提供）
5. 无 funding rate / open interest / liquidation data —— 留 v0.x perpetual 接入
